## Hardware quickstart

Get your VaultDrive initialized and ready to use in under 5 minutes.

### What you need

- VaultDrive device (any model)
- Computer with USB 3.0 or higher port
- VaultWares Manager (optional, for advanced features)

### Initialize your VaultDrive

<Steps>
  <Step title="Connect the device">
    Plug the VaultDrive into an available USB-C or USB-A port. The status LED will flash amber, indicating the device is waiting for initialization.

    <Note>
      On first use, the device mounts a small read-only partition containing the setup utility. Do not attempt to format this partition.
    </Note>
  </Step>

  <Step title="Launch the setup utility">
    The device auto-runs the setup utility on supported systems. Alternatively, open the mounted volume and run:

    <Tabs>
      <Tab title="Windows">
        ```cmd
        E:\VaultWares-Setup.exe
        ```
      </Tab>
      <Tab title="macOS">
        ```bash
        open /Volumes/VAULTDRIVE/VaultWares-Setup.app
        ```
      </Tab>
      <Tab title="Linux">
        ```bash
        /media/$USER/VAULTDRIVE/vaultdrive-setup
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Set your master password">
    When prompted, create a strong master password. VaultWares enforces these minimum requirements by default:

    - At least 8 characters (12+ recommended)
    - At least one uppercase letter
    - At least one number
    - At least one special character

    <Warning>
      Your master password cannot be recovered by VaultWares. If you forget it, the encrypted data cannot be accessed. Store your password in a secure password manager.
    </Warning>
  </Step>

  <Step title="Enroll biometrics (biometric models only)">
    For VaultDrive Biometric, you will be prompted to scan your fingerprint three times to complete enrollment. Up to 10 fingerprints can be enrolled for shared-device scenarios.
  </Step>

  <Step title="Access your encrypted volume">
    After successful authentication, the encrypted volume mounts automatically:

    | OS | Mount location |
    |----|---------------|
    | Windows | Assigned drive letter (e.g., `E:\`) |
    | macOS | `/Volumes/VaultDrive` |
    | Linux | `/media/$USER/VaultDrive` |

    The status LED turns solid green when the volume is unlocked and mounted.
  </Step>
</Steps>

### CLI management (optional)

Install VaultWares Manager for command-line management:

```bash
# Install VaultWares Manager
# macOS/Linux
curl -sSL https://get.vaultwares.com/manager | bash

# Windows (PowerShell)
iwr https://get.vaultwares.com/manager/install.ps1 | iex

# Verify installation
vw-storage --version

# Check connected devices
vw-storage list-devices

# Change password
vw-storage change-password --device /dev/sdb

# Enable auto-lock after 5 minutes of inactivity
vw-storage set-policy --device /dev/sdb --auto-lock 5
```

## Software quickstart

Install VaultCrypt and encrypt your first volume in under 10 minutes.

### System requirements

| Component | Minimum | Recommended |
|-----------|---------|-------------|
| OS | Windows 10 / macOS 11 / Ubuntu 20.04 | Windows 11 / macOS 13 / Ubuntu 22.04 |
| CPU | 2-core x86-64 | 4-core x86-64 or Apple Silicon |
| RAM | 4 GB | 8 GB |
| Disk space | 500 MB | 1 GB |
| .NET / JRE | .NET 6.0+ | .NET 8.0+ |

### Install and activate VaultCrypt

<Steps>
  <Step title="Download the installer">
    Download VaultCrypt from the [customer portal](https://portal.vaultwares.com/downloads) or use the CLI:

    ```bash
    # Download latest stable release
    curl -O https://releases.vaultwares.com/vaultcrypt/latest/vaultcrypt-installer.sh
    chmod +x vaultcrypt-installer.sh
    sudo ./vaultcrypt-installer.sh
    ```

    <Tip>
      Enterprise customers can also deploy VaultCrypt silently via MSI, PKG, or DEB/RPM packages. See the [Software deployment guide](/installation/software-deployment).
    </Tip>
  </Step>

  <Step title="Activate your license">
    After installation, activate VaultCrypt with your license key:

    <Tabs>
      <Tab title="GUI">
        Launch VaultCrypt. When prompted, enter your license key and click **Activate**. An internet connection is required for online activation.
      </Tab>
      <Tab title="CLI">
        ```bash
        # Online activation
        vaultcrypt activate --key VWCR-XXXX-XXXX-XXXX-XXXX

        # Offline activation (air-gapped systems)
        vaultcrypt activate --key VWCR-XXXX-XXXX-XXXX-XXXX --offline
        # Follow the on-screen instructions to generate an activation request file
        # and submit it at https://portal.vaultwares.com/offline-activation
        ```
      </Tab>
    </Tabs>

    A successful activation prints:
    ```
    License activated successfully.
    Edition:    Enterprise
    Valid until: 2026-12-31
    Seats:       250 / 500 used
    ```
  </Step>

  <Step title="Encrypt your first volume">
    Choose between full-disk encryption or container-based encryption:

    <Tabs>
      <Tab title="Full-disk encryption">
        ```bash
        # Encrypt an entire disk partition
        vaultcrypt encrypt-disk \
          --device /dev/sdc \
          --algorithm AES-256-XTS \
          --key-source password

        # Or encrypt an existing Windows drive
        vaultcrypt encrypt-disk --drive D: --algorithm AES-256-XTS
        ```

        <Warning>
          Full-disk encryption formats the target partition. Back up your data before proceeding.
        </Warning>
      </Tab>
      <Tab title="Encrypted container">
        ```bash
        # Create a 10 GB encrypted container file
        vaultcrypt create-container \
          --output ~/secure-vault.vc \
          --size 10G \
          --algorithm AES-256-XTS \
          --key-source password

        # Mount the container
        vaultcrypt mount ~/secure-vault.vc --mountpoint ~/vault

        # Unmount when done
        vaultcrypt unmount ~/vault
        ```
      </Tab>
    </Tabs>
  </Step>

  <Step title="Verify encryption status">
    ```bash
    # Check encryption status for all managed volumes
    vaultcrypt status

    # Sample output:
    # Volume             Algorithm      Status     Mounted
    # /dev/sdc           AES-256-XTS    Encrypted  Yes
    # ~/secure-vault.vc  AES-256-XTS    Encrypted  Yes
    ```
  </Step>

  <Step title="Configure automatic mounting (optional)">
    For servers and headless systems, configure VaultCrypt to mount volumes automatically on boot:

    ```bash
    # Register a volume for auto-mount using HSM key
    vaultcrypt auto-mount \
      --volume /dev/sdc \
      --key-source hsm \
      --hsm-slot 1 \
      --at-boot

    # Or use a key file stored in a secure location
    vaultcrypt auto-mount \
      --volume /dev/sdc \
      --key-file /etc/vaultcrypt/keys/sdc.key \
      --at-boot
    ```
  </Step>
</Steps>
