# Command-Line Tool (`vw`)

To streamline and unify the vast collection of PowerShell automation, synchronization, and media-processing scripts across the VaultWares workspace, all useful utilities have been consolidated under a single command-line interface called `vw`.

The `vw` utility operates globally, routing parameters down to modular scripts located in various repositories across the workspace.

---

## Global Access

The CLI is exposed as the global `vw` command. It is implemented as a batch wrapper (`vw.bat`) pointing to the PowerShell router (`vw.ps1`).

```bash
vw <command> [arguments]
```

To view the main help menu, run:

```bash
vw --help
```

## Command Registry

Commands are grouped into logical operational categories:

### Workspace Management
- **`sync`**: Run bulk Git pull and workspace synchronization across repository layers.
- **`shortcuts`**: Install or refresh Desktop shortcuts for standard tool configurations.
- **`rebuild`**: Clean and rebuild python packages, node modules, and dependencies.
- **`recreate-venvs`**: Reinitialize virtual environments (`.venv`) across workspaces.
- **`launch-team`**: Dispatch team of supervisor and worker AI agents.
- **`deploy-dispatch`**: Bootstrap VaultWares dispatch server directories.
- **`backup-task`**: Create a scheduled task to run the Auto Backup script silently every 5 minutes.
- **`swap-sfw`**: Toggle and hide/unhide directories based on IsSFW environment variable.
- **`find-scripts`**: Search workspace and output script file listings to the Desktop.

### Git Automation
- **`git-pull-all`**: Iterate through workspace and pull updates for submodules and branches.
- **`git-auto`**: Execute repository checks and send email alerts on remote changes.

### Agent Ledger
- **`archive-ledger`**: Archive historical ledger entries to optimize performance.
- **`record-change`**: Log an agent transaction or changes to the Shared Agent Ledger database.
- **`render-ledger`**: Generate detailed HTML/Markdown reports of agent activities.
- **`render-impact`**: Compile work metrics and update visual workspace progression indicators.
- **`sync-ledger`**: Sync local ledger records to central database endpoints.
- **`start-tracker`**: Launch the keyboard/mouse idle telemetry service in the background.

#### Input Tracker restart policy

Operational note, Thu, 02 Jul 2026 16:45: the Input Tracker is a resident Python process (`agent-ledger/scripts/track-input.py`). Updating or pulling `agent-ledger` does not reload that process by itself, so an old tracker can keep running stale code even when the scheduled task appears healthy.

The durable fix is to run the tracker through `agent-ledger/scripts/start-input-tracker.ps1` and register `VaultWares-InputTracker` with `MultipleInstances = Parallel`. The launcher stops any existing `track-input.py` process for the same script path before starting the new one. This makes `Start-ScheduledTask -TaskName VaultWares-InputTracker` a real code reload instead of a no-op.

After any tracker code change or repo sync, verify with:

```powershell
Start-ScheduledTask -TaskName "VaultWares-InputTracker"
Start-Sleep -Seconds 70
Get-Content "C:\Users\Administrator\Desktop\Github Repos\agent-ledger\input-state\input-tracker-health.json"
```

The health payload should show a fresh `updated_at`, a current `pid`, `status` as `online` or `listening`, and a `last_post_at` after the restart once activity has flushed.

## Addition Protocol

To extend the global `vw` CLI with a new automation script, follow these standard steps:

<Steps>
  <Step title="Place the script in the workspace">
    Save the script in the most relevant sub-repository (e.g. `vault-explorer/scripts/pwsh/My-New-Script.ps1` or `agent-ledger/scripts/My-New-Script.ps1`). Ensure the script is modular, handles parameters correctly, and fails gracefully with non-zero exit codes on error.
  </Step>

  <Step title="Register the command">
    Open the registry file `C:\Users\Administrator\Desktop\pwsh\VaultWares CLI\vw-commands.ps1` and add a new entry to the `$Commands` hashtable:

    ```powershell
    "my-new-command" = @{
        ScriptPath  = "$RepoRoot\path\to\your\My-New-Script.ps1"
        Description = "Brief description of the tool's behavior."
        Category    = "Workspace Management" # Or another category
    }
    ```
  </Step>

  <Step title="Test the integration">
    Verify the command appears correctly in the CLI help menu and runs successfully:

    ```bash
    # Show CLI help to check categorization
    vw --help

    # Test run
    vw my-new-command --parameter value
    ```
  </Step>

  <Step title="Document the command">
    Update this document (`docs-content/operations/command-line-tool.mdx`) under the appropriate category section to keep developer resources synced.
  </Step>
</Steps>

<Info>
  Always verify that the target script works inside non-interactive shells and does not wait on user input unless explicitly prompt-driven.
</Info>
