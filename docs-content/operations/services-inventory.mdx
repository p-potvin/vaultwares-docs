# Services inventory (Private)

This page is a **single source of truth table** for “what we run”.

It exists so that:

- a non-technical person can understand *what each URL is*

> If you add a new public URL, update this table the same day.

- `operations/health-ledger` (service health, Probe Joker, Alarm Joker, Token Joker)

## Deployment and alert note

Every repo in this table must follow the same control-plane model:

- GitHub is only an event source (webhook)
- deploy/test/probe jobs happen inside the tailnet on the configured project
  target host
- **no GitHub Actions deploys**
- alert severity is determined by the Health Ledger alert profile, not by the
  host that receives the webhook
- local Ollama/Gemma remediation is allowed only for explicitly permitted
  non-production Low flows
- `vaultwares-api` owns stats/ledger telemetry DB access, batching ingest,
  auth, normalized monitor APIs, and future business rules. Static dashboards
  such as `stats.vaultwares.ca` must not read source ledger files or Postgres
  directly.
- VaultWares apps pass request correlation IDs as the `correlationId` query
  parameter, not as the primary header contract. VaultWares-generated IDs use
  `vw_{3-or-4-char Jira code}_c{7 hex chars}`, for example
  `vw_API_c123abc4`; non-VaultWares products may use their own prefix, for
  example `pk_PKT_c123456h`. The same correlation ID should persist from
  client request creation until the request completes or fails at the API
  boundary. Streams, downloads, and other continuous connections may create a
  child ID for the long-lived transfer.
- `vaultwares-api` request logging defaults to important events only: blocked
  requests, crashes, HTTP errors, and slow requests. Routine successful calls
  are intentionally suppressed to keep Kiwi volume usable. API syslog records
  must include source/client context in addition to the VPS proxy peer address.
- Vault-flows ComfyUI jobs submitted through `vaultwares-api` are serialized:
  one ComfyUI workflow job may execute at a time, regardless of source IP.
  Asynchronous queue clients can provide a completion callback URL; callbacks
  are limited to loopback, private, or tailnet hosts unless explicitly allowed.

Source of truth: `operations/deployment-flow`.

## Ops surfaces on `greencloud-vps` (not user-facing)

| Surface | Listener / unit | Role |
|---|---|---|
| `vw-webhookd` | systemd service, `127.0.0.1:9033` | Verifies GitHub webhook signatures + dispatches per-repo deploy scripts. Runs as `vwdeploy`. |
| `vaultwares-hooks` | systemd service, `127.0.0.1:8787` | Tailnet-only webhook handler (uvicorn). |
| `vw-secretsd` | systemd service, `127.0.0.1:9444` | Local secrets shim behind `warden.vaultwares.ca`. |
| `health-ledger-probe.service` | systemd service | Active Probe Joker loop under `/opt/health-ledger`; writes Health Ledger events and rollups. |
| `health-ledger-dashboard.service` | systemd service, `127.0.0.1:8790` | First-party Health Ledger dashboard; reads rollups/events and does not run probes. |
| `health-ledger-alarm.service` | systemd service, disabled until `/etc/health-ledger/alarm.env` is configured | Alarm Joker notification loop for incident locks and SMS/email/desktop fanout. |
| `vw-deny-watch.timer` | systemd timer, disabled 2026-06-03 | Legacy GitHub issue writer for webhook `bad_signature` clusters. Replaced by Health Ledger direction. |
| `vw-token-expiry-watch.timer` | systemd timer, disabled 2026-06-03 | Legacy GitHub issue writer for token expiry. Replaced by Token Joker direction. |

## Notes on the table above

- `fullxxx.video` and `prom-king.xyz` are **WordPress installs** under
  `/var/www/<site>/public/`. Their front-door/prelanding behavior and their
  tube plugins are tracked separately in Health Ledger so the repo/runtime
  ownership is explicit. The plugins driving the catalogs live under
  `wp-content/plugins/{fullxxx-video,promking-tube,tube-shared}` and are
  rsynced from `/var/www/tube-sites-src/` by the deploy script.
- The GitHub org **is `Prom-King`** (capital `P`, capital `K`, hyphen) for
  every repo in this list — the lowercase form `prom-king/...` won't match.
