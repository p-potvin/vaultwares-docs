# Services inventory (Private)

This page is a **single source of truth table** for “what we run”.

It exists so that:

- a non-technical person can understand *what each URL is*
- an assistant can quickly map a URL → machine → repo → deployment flow

> If you add a new public URL, update this table the same day.

Related:
- `operations/deployment-flow` (canonical deployment flow)
- `operations/health-ledger` (service health, Probe Joker, Alarm Joker, Token Joker)

## Deployment and alert note

Every repo in this table must follow the same control-plane model:

- GitHub is only an event source (webhook)
- deploy/test/probe jobs happen inside the tailnet on the configured project
  target host
- **no GitHub Actions deploys**
- alert severity is determined by the Health Ledger alert profile, not by the
  host that receives the webhook
- local Ollama/Gemma remediation is allowed only for explicitly permitted
  non-production Low flows

Source of truth: `operations/deployment-flow`.

| `vw-token-expiry-watch.timer` | systemd timer, disabled 2026-06-03 | Legacy GitHub issue writer for token expiry. Replaced by Token Joker direction. |

## Notes on the table above

- `fullxxx.video` and `prom-king.xyz` are **WordPress installs** under
  `/var/www/<site>/public/`. Their front-door/prelanding behavior and their
  tube plugins are tracked separately in Health Ledger so the repo/runtime
  ownership is explicit. The plugins driving the catalogs live under
  `wp-content/plugins/{fullxxx-video,promking-tube,tube-shared}` and are
  rsynced from `/var/www/tube-sites-src/` by the deploy script.
- The GitHub org **is `Prom-King`** (capital `P`, capital `K`, hyphen) for
  every repo in this list — the lowercase form `prom-king/...` won't match.
