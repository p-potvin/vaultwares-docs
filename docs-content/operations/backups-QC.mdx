# Sauvegardes (Privé)

Les sauvegardes horaires s'exécutent sur `greencloud-vps` (`vaultwares-1`) en utilisant `rclone` vers votre Google Drive.

## Ce qui est sauvegardé

Le job horaire crée un fichier `.tgz` contenant (inclut des secrets) :

- `/etc/nginx` (sites + nginx.conf)
- `/etc/systemd/system` (unités/minuteries personnalisées)
- `/etc/vw-secretsd` (Configuration VaultWarden)
- `/etc/vw-webhookd` (Configuration du récepteur de webhooks)
- `/etc/letsencrypt` (Clés TLS + configuration de renouvellement)
- `/opt/actions-runner` (Configuration du runner GitHub auto-hébergé)
- `/opt/vw-jira-sync`
- `/opt/vault-warden`
- `/opt/vaultwares-webhooks`

## Mode de stockage Google Drive

Sun, 28 Jun 2026 15:57 America/Toronto : les scripts de sauvegarde actifs écrivent directement dans le remote Google Drive non chiffré :

```bash
vw_gdrive:VaultWares/backups
```

Ne ciblez pas `vw_gdrive_crypt:` pour les nouvelles écritures de sauvegarde. L'ancien remote crypt peut rester dans `/etc/vw-backup/rclone.conf` seulement pour récupérer d'anciennes archives chiffrées au besoin.

Les anciennes clés crypt sont stockées dans VaultWarden comme :

- `vw_rclone_crypt_password`
- `vw_rclone_crypt_password2`

Ne collez jamais ces valeurs dans des documents, des problèmes ou des discussions.

## Où ça s'exécute

- Script : `/usr/local/bin/vw-backup.sh`
- Unité systemd : `/etc/systemd/system/vw-backup.service`
- Minuterie : `/etc/systemd/system/vw-backup.timer` (`OnCalendar=hourly`)
- Configuration rclone : `/etc/vw-backup/rclone.conf` (root seulement)
- Journal : `/var/log/vaultwares/vw-backup.log`

## Autorisation unique Google Drive

L'OAuth Google Drive doit être complété une seule fois sur le VPS (flux sans tête) :

```bash
sudo rclone --config /etc/vw-backup/rclone.conf config reconnect vw_gdrive:
```

Vérifiez ensuite :

```bash
sudo rclone --config /etc/vw-backup/rclone.conf lsd vw_gdrive:
sudo rclone --config /etc/vw-backup/rclone.conf lsd vw_gdrive:VaultWares/backups
```

## Contrôle / Dépannage

```bash
sudo systemctl status vw-backup.timer
sudo systemctl status vw-backup.service
sudo journalctl -u vw-backup.service -n 200 --no-pager
tail -n 200 /var/log/vaultwares/vw-backup.log
```

Si l'OAuth est manquant, `vw-backup.service` échoue rapidement avec une erreur vous indiquant de lancer `config reconnect`.

## Restauration (vue d'ensemble)

1. Téléchargez le fichier `.tgz` souhaité depuis `vw_gdrive:VaultWares/backups/<host>/` (le nom du dossier hôte dépend du nom d'hôte).
2. Extrayez-le dans un répertoire de staging.
3. Restaurez les chemins sélectionnés (nginx/systemd/letsencrypt) puis rechargez nginx/systemd.

Pour les anciennes archives écrites via `vw_gdrive_crypt:`, lisez-les à travers cet ancien remote pour que rclone les déchiffre avant la restauration.
