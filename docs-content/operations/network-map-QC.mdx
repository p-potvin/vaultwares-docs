# Carte réseau (Privé)

## Comment lire cette page (super simple)

Imagine 3 “endroits” :

1. **Internet (public)** : la grande rue. N’importe qui peut frapper à la porte.
2. **Tailnet (privé)** : un corridor privé. Juste les appareils autorisés peuvent entrer.
3. **Machines** : nos ordinateurs (PC + serveurs) qui font rouler les services.

```mermaid
flowchart LR
  Public["Internet (public)"] -->|HTTPS 443| VPS["greencloud-vps\n(le serveur public)"]
  Tailnet["Tailnet (privé)\nTailscale"] <--> VPS
  Tailnet <--> PC["Clopeux-Desktop\n(le PC)"]
```

<Tip>
  Si tu montres ça à un public non-tech : garde juste l’idée “Internet = public, Tailnet = corridor privé”.
</Tip>

## Modèle d’accès

- **Chiffrement réseau:** Tailscale (WireGuard).
- **TLS public:** nginx + Let’s Encrypt sur les VPS.
- **SSH:** **Seulement sur le Tailnet** (port `22` autorisé uniquement sur `tailscale0`).
- **RDP:** Seulement sur le Tailnet vers `Clopeux-Desktop:3389` (contrôlé par la politique).

## Liste d’intégration

1. Rejoindre le tailnet (client Tailscale).
2. Confirmer que votre appareil apparaît dans la console d'administration et est autorisé.
3. Lire cette page et le document de politique Tailscale.
4. Obtenir l'accès à VaultWarden (`https://warden.vaultwares.ca`) (liste blanche tailnet + autorisation de service).
5. Vérifier :
   - Vous pouvez atteindre `docs.vaultwares.ca` depuis le tailnet.
   - Vous pouvez atteindre `vaultwares.ca` publiquement.
   - Vous pouvez atteindre le point d'accès API VaultWares nécessaire pour votre travail (si la politique le permet).
