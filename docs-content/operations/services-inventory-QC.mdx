# Inventaire des services (Privé)

Cette page est une **table source de vérité** pour répondre à : “Qu’est-ce qu’on roule, pis où?”

Elle sert à :

- aider quelqu’un de non-tech à comprendre *c’est quoi chaque URL*
- aider un assistant à faire le lien **URL → machine → repo → déploiement**

> Si on ajoute une nouvelle URL publique, on met à jour cette table la même journée.

## Note de déploiement (pareil pour tout le monde)

Chaque repo dans cette table doit suivre **le même modèle de déploiement** :

- GitHub sert juste à envoyer l’événement (webhook)
- le déploiement se fait dans le tailnet sur `greencloud-vps`
- **aucun déploiement via GitHub Actions**

Source de vérité : `operations/deployment-flow`.

## Surfaces média tailnet sur `vps-ovhcloud`

Ces services sont destinés aux opérateurs dans le tailnet. Les services Docker
vivent sur `vps-ovhcloud` dans `/opt/vw-media-stack/docker-compose.yml`, avec
la config et les données persistantes sous `/srv/vw-media/`. `greencloud-vps`
répond aux noms privés via `dnsmasq`, puis nginx proxy vers OVH sur le tailnet.

Greencloud sert les surfaces média avec le certificat SAN
`/etc/letsencrypt/live/media.vaultwares.ca/`. Les autres noms privés
(`docs.vaultwares.ca`, `hooks.vaultwares.ca`, `warden.vaultwares.ca`,
`monitor.vaultwares.ca`, `stats.vaultwares.ca`, `mcp.vaultwares.ca`, etc.)
doivent garder des `listen 100.73.93.84:443 ssl;` explicites. Sans ces listeners,
nginx peut choisir le certificat média pour les requêtes qui arrivent par le
tailnet.

## Notes sur Prom-King / FullXXX

- `fullxxx.video` et `prom-king.xyz` sont des **installations WordPress** sous
  `/var/www/<site>/public/`.
- Leur front door/prelanding et leurs plugins tube sont suivis séparément dans
  Health Ledger pour rendre l'ownership repo/runtime explicite.
- Les plugins qui alimentent les catalogues vivent sous
  `wp-content/plugins/{fullxxx-video,promking-tube,tube-shared}` et sont
  rsynced depuis `/var/www/tube-sites-src/` par le script de déploiement.
- L'org GitHub est **`Prom-King`** pour ces repos. La forme minuscule
  `prom-king/...` ne doit pas être utilisée comme nom de repo.
