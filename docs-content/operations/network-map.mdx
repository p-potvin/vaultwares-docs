## Access model / Modele d'acces

- **Network encryption:** Tailscale (WireGuard).
- **Public TLS:** nginx + Let's Encrypt on VPSes.
- **SSH:** **Tailnet-only** (port `22` allowed only on `tailscale0`).

### SSH commands / Commandes SSH

```powershell
# OVH / OVHCloud / vps-ovhcloud (user: ubuntu, key: tube-site-vps)
ssh -i "$env:USERPROFILE\.ssh\tube-site-vps" ubuntu@100.67.25.118

# GreenCloud VPS (user: root, key: greencloud-vps)
ssh -i "$env:USERPROFILE\.ssh\greencloud-vps" root@100.73.93.84
```

> Do not use `root@100.67.25.118` for normal diagnostics on OVH; that account rejects the local operator key. Use `ubuntu@` instead.

- **RDP:** `Clopeux-Desktop:3389` is restricted to the operator's other PC.
  VPS hosts such as `vps-ovhcloud` and `greencloud-vps` should not be able to
  open RDP sessions to Clopeux.

## Onboarding checklist / Liste d'integration

1. Join the tailnet (Tailscale client).
2. Confirm your device appears in the admin console and is authorized.
3. Read this page and the Tailscale policy doc.
4. Get access to VaultWarden (`https://warden.vaultwares.ca`) (tailnet allowlist + service authz).
5. Verify:
   - You can reach `docs.vaultwares.ca` from tailnet.
   - You can reach `vaultwares.ca` publicly.
   - You can reach the VaultWares API endpoint needed for your work (policy permitting).
