<Info>
  **Prerequisites**:
  - Node.js 20+ (Node 22 recommended)
  - `docs-content/` source pages with YAML frontmatter
</Info>

Follow these steps to install and run VaultWares Docs on your operating system.

<Steps>
<Step title="Install dependencies">

```bash
npm ci
```
</Step>

<Step title="Generate page resources">

The frontend consumes generated locale resources under `src/resources/`. Regenerate them after editing `docs-content/`:

```bash
npm run generate:page-resources
```

</Step>
</Steps>

## Preview locally

Run the dev server:

```bash
npm run dev
```

Vite uses port 5173 by default and will pick the next available port if it is busy. You can force a port:

```bash
npm run dev -- --port 3333
```

## Build and preview

Build:

```bash
npm run build
```

Preview the production build locally:

```bash
npm run preview
```

## Code formatting

We suggest using extensions on your IDE to recognize and format MDX. If you're a VSCode user, consider the [MDX VSCode extension](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) for syntax highlighting, and [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) for code formatting.

## Troubleshooting

<AccordionGroup>
  <Accordion title='Issue: Node version mismatch'>

    This is usually a Node version mismatch. Use Node 20+ (Node 22 recommended) and reinstall:
    1. Delete `node_modules/` and the lockfile only if needed
    2. Run `npm ci`
  </Accordion>

  <Accordion title="Issue: Content looks stale after editing docs-content/">

    Solution: Regenerate resources and restart the dev server:
    1. `npm run generate:page-resources`
    2. Restart `npm run dev`
  </Accordion>
</AccordionGroup>
