## Aperçu

Utilisez ce guide pour diagnostiquer et résoudre les problèmes courants avec le matériel et le logiciel VaultWares. Si un problème n'est pas couvert ici, consultez le [portail de support](https://portal.vaultwares.com/support) ou contactez [support@vaultwares.com](mailto:support@vaultwares.com).

## Commandes de diagnostic

Exécutez ces commandes d'abord pour recueillir des informations pour toute session de dépannage :

```bash
# System-wide health check
vw-admin health-check --full --output /tmp/health-$(date +%Y%m%d%H%M).json

# Collect a diagnostic bundle (for support escalation)
vw-admin diag collect \
  --include-logs 48h \
  --include-config true \
  --output /tmp/diag-$(hostname)-$(date +%Y%m%d%H%M).tar.gz

# Check service status
vw-admin service status --all
```

## Dépannage du stockage chiffré

<AccordionGroup>
  <Accordion title="Le périphérique n'est pas reconnu par le système d'exploitation">
    **Symptômes**: VaultDrive n'apparaît pas comme un lecteur dans l'Explorateur de fichiers / Finder / `lsblk`.

    **Diagnostic**:
    ```bash
    # Check if the device is seen by the USB subsystem
    lsusb | grep -i vault
    dmesg | tail -20
    ```

    **Étapes de résolution**:
    1. Essayez un autre port USB (doit être USB 3.0 ou supérieur)
    2. Essayez le périphérique sur un autre ordinateur pour isoler le problème
    3. Sur Linux, vérifiez les conflits de modules : `sudo modprobe uas`
    4. Sur Windows, mettez à jour les pilotes USB dans le Gestionnaire de périphériques
    5. Inspectez le connecteur USB-C pour détecter des dommages physiques
    6. Si aucune des étapes ci-dessus ne fonctionne, [déposez une demande de RMA](/support/rma)
  </Accordion>

  <Accordion title="Échec d'authentification constant (mot de passe correct)">
    **Symptômes**: La saisie du mot de passe est acceptée, mais l'authentification échoue de manière répétée.

    **Diagnostic**:
    ```bash
    vw-storage device-info --device /dev/sdb
    # Check: failed_attempts, lockout_status, firmware_version
    ```

    **Étapes de résolution**:
    1. Assurez-vous que la touche Verr. Maj n'est pas activée
    2. Vérifiez si le périphérique est en état de verrouillage : recherchez une LED rouge clignotante
    3. Si verrouillé, attendez la durée de verrouillage (par défaut 30 minutes) ou contactez le support avec une preuve de propriété
    4. Vérifiez si une politique d'entreprise annule le mot de passe — les périphériques gérés par politique peuvent nécessiter un code PIN administrateur
    5. Mettez à jour le micrologiciel du périphérique : `vw-storage firmware-update --device /dev/sdb`
  </Accordion>

  <Accordion title="Vitesse de transfert lente">
    **Symptômes**: Les vitesses de transfert sont significativement inférieures aux spécifications nominales.

    **Diagnostic**:
    ```bash
    # Check USB negotiation speed
    udevadm info -a -n /dev/sdb | grep speed

    # Benchmark the device
    vw-storage benchmark --device /dev/sdb
    ```

    **Étapes de résolution**:
    1. Confirmez que vous utilisez un port USB 3.0+ (couleur bleue ou logo SuperSpeed)
    2. Utilisez le câble USB-C inclus; les câbles tiers peuvent limiter les vitesses
    3. Désactivez temporairement l'analyse antivirus en temps réel sur le chemin de destination pour tester
    4. Assurez-vous que l'ordinateur hôte n'est pas en mode d'économie d'énergie qui bride l'USB
    5. Reformatez le périphérique (note : cela efface toutes les données) pour résoudre la fragmentation du système de fichiers
  </Accordion>

  <Accordion title="Le capteur biométrique n'enregistre pas l'empreinte digitale">
    **Symptômes**: L'enregistrement de l'empreinte digitale échoue après plusieurs tentatives.

    **Étapes de résolution**:
    1. Nettoyez le capteur avec un chiffon sec et non pelucheux
    2. Assurez-vous que votre doigt est sec et exempt de coupures ou d'abrasions
    3. Essayez un autre doigt
    4. Mettez à jour le micrologiciel : `vw-storage firmware-update --device /dev/sdb`
    5. Réinitialisez le module biométrique : `vw-storage biometric-reset --device /dev/sdb` (nécessite un code PIN administrateur)
  </Accordion>
</AccordionGroup>

## Dépannage du HSM

<AccordionGroup>
  <Accordion title="Le HSM n'est pas détecté après l'installation">
    **Symptômes**: `vw-hsm status` signale qu'aucun périphérique n'est trouvé.

    **Diagnostic**:
    ```bash
    # PCIe models
    lspci | grep -i vault
    lsmod | grep vaulthsm
    journalctl -u vaulthsm --no-pager -n 50

    # Network models
    ping hsm01.corp.example.com
    curl -k https://hsm01.corp.example.com:8443/api/status
    ```

    **Résolution**:
    ```bash
    # Reinstall driver (PCIe)
    sudo systemctl stop vaulthsm
    sudo rmmod vaulthsm_pcie
    sudo modprobe vaulthsm_pcie
    sudo systemctl start vaulthsm

    # Check network reachability (Network models)
    vw-hsm network-test --host hsm01.corp.example.com --port 1792
    ```
  </Accordion>

  <Accordion title="PKCS#11 CKR_TOKEN_NOT_PRESENT">
    **Symptômes**: Les applications signalent `CKR_TOKEN_NOT_PRESENT` ou `CKR_SLOT_ID_INVALID`.

    **Résolution**:
    ```bash
    # List available tokens
    vw-hsm slot list

    # Verify PKCS#11 library path
    echo $PKCS11_MODULE
    ls -la /usr/lib/vaulthsm-pkcs11.so

    # Test PKCS#11 with pkcs11-tool
    pkcs11-tool --module /usr/lib/vaulthsm-pkcs11.so --list-slots
    ```
  </Accordion>

  <Accordion title="Latence d'opération élevée">
    **Symptômes**: Les opérations cryptographiques prennent significativement plus de temps que prévu.

    **Diagnostic**:
    ```bash
    # Check HSM performance metrics
    vw-hsm monitor --metrics latency,queue-depth,errors --interval 5s

    # Check host resource utilization
    top -bn1 | grep -E "Cpu|Mem"

    # Review HSM queue depth
    vw-hsm status --verbose | grep queue
    ```

    **Résolution**:
    1. Si la profondeur de la file d'attente est élevée (>100), envisagez d'ajouter des HSM au cluster
    2. Vérifiez la latence réseau entre le client et le HSM réseau : `vw-hsm network-test --latency`
    3. Vérifiez les conflits de ressources sur l'hôte HSM (modèles PCIe)
    4. Examinez les modèles d'opération — assurez-vous que les opérations en vrac sont regroupées de manière appropriée
  </Accordion>

  <Accordion title="Échec de synchronisation du cluster">
    **Résolution**:
    ```bash
    # Check cluster status
    vw-hsm cluster status

    # Check NTP synchronization (clock skew can break cluster sync)
    timedatectl status
    chronyc tracking

    # Force resynchronization
    vw-hsm cluster sync --force --from hsm01

    # Review cluster logs
    vw-hsm logs --component cluster --level debug --last 100
    ```
  </Accordion>
</AccordionGroup>

## Dépannage des dispositifs biométriques

<AccordionGroup>
  <Accordion title="VaultScan ne se connecte pas au serveur d'enregistrement">
    **Diagnostic**:
    ```bash
    vw-biometric test-connection \
      --device 192.168.1.51 \
      --server https://vaultaccess.corp.example.com
    ```

    **Résolution**:
    1. Vérifiez que l'URL et le port du serveur sont accessibles depuis le sous-réseau du périphérique
    2. Vérifiez la validité du certificat TLS (un certificat expiré est une cause fréquente)
    3. Confirmez que le périphérique dispose d'une résolution DNS correcte : vérifiez via le menu LCD → Réseau → Test DNS
    4. Examinez les exigences de pare-feu de VaultScan : TCP 443 sortant vers le serveur d'enregistrement
  </Accordion>

  <Accordion title="Taux de rejet erroné trop élevé">
    **Résolution**:
    1. Augmentez la sensibilité du seuil de correspondance : `vw-biometric set-threshold --device 192.168.1.51 --threshold medium` (options : low, medium, high, very-high)
    2. Réenregistrez les utilisateurs avec de meilleurs échantillons de qualité
    3. Nettoyez les surfaces du scanner
    4. Assurez un éclairage adéquat pour les modules de reconnaissance faciale
  </Accordion>
</AccordionGroup>

## Dépannage logiciel

<AccordionGroup>
  <Accordion title="Le volume VaultCrypt ne monte pas après un redémarrage">
    **Symptômes**: Le volume chiffré ne monte pas automatiquement après un redémarrage du système.

    **Diagnostic**:
    ```bash
    vaultcrypt status
    journalctl -u vaultcrypt --no-pager -n 50
    ```

    **Résolution**:
    ```bash
    # Check auto-mount configuration
    vaultcrypt auto-mount list

    # Re-register auto-mount entry
    vaultcrypt auto-mount \
      --volume /dev/sdc \
      --key-source hsm \
      --hsm-slot 1 \
      --at-boot

    # Verify HSM connectivity (required for HSM-keyed volumes)
    vw-hsm status
    ```
  </Accordion>

  <Accordion title="Synchronisation LDAP de VaultAccess échoue">
    **Diagnostic**:
    ```bash
    vw-admin ad sync --dry-run --verbose
    vw-admin ldap test --verbose
    ```

    **Résolution**:
    1. Vérifiez que le mot de passe du compte de service n'a pas expiré
    2. Confirmez la connectivité réseau : `telnet dc01.corp.example.com 389`
    3. Vérifiez les problèmes de certificat LDAP si vous utilisez LDAPS : `openssl s_client -connect dc01.corp.example.com:636`
    4. Examinez le filtre de recherche LDAP pour détecter des erreurs de syntaxe
  </Accordion>

  <Accordion title="Les tâches VaultBackup échouent constamment">
    **Diagnostic**:
    ```bash
    vw-backup jobs get --id <job-id> --verbose
    vw-backup logs --job <job-id> --level debug
    ```

    **Causes courantes**:
    - Répertoire de sauvegarde plein : `vw-backup repo stats --name local-nas`
    - HSM inaccessible pour l'enrobage de clés : `vw-hsm status`
    - Permissions insuffisantes sur le chemin de destination
    - Délai réseau pour la réplication hors site : vérifiez la bande passante et la connectivité
  </Accordion>
</AccordionGroup>

## Emplacements des journaux

| Produit | Fichier de journal / commande |
|---------|-------------------|
| VaultCrypt | `/var/log/vaultcrypt/vaultcrypt.log` |
| VaultAccess | `/var/log/vaultaccess/vaultaccess.log` |
| VaultBackup | `/var/log/vaultbackup/vaultbackup.log` |

| VaultHSM driver | `journalctl -u vaulthsm` |
| VaultHSM audit | `vw-hsm audit-query --last 100` |

| VaultScan | `vw-biometric logs --device 192.168.1.51` |
| VaultGate | `vaultgate logs --component all --last 500` |

## Procédure d'escalade

Si vous ne parvenez pas à résoudre un problème à l'aide de ce guide :

<Steps>
  <Step title="Collecter les diagnostics">
    ```bash
    vw-admin diag collect \
      --include-logs 72h \
      --include-config true \
      --output /tmp/diag-$(hostname)-$(date +%Y%m%d%H%M).tar.gz
    ```
  </Step>
  <Step title="Ouvrir un ticket de support">
    Connectez-vous à [portal.vaultwares.com/support](https://portal.vaultwares.com/support) et créez un nouveau ticket. Incluez :
    - Description détaillée du problème et étapes de reproduction
    - Le paquet de diagnostic de l'étape précédente
    - Versions des produits affectés : `vw-admin version --all`
    - Urgence (de P1 panne de production à P4 question)
  </Step>
  <Step title="Support d'urgence">
    Pour les pannes de production P1, appelez la ligne de support d'urgence 24/7 indiquée dans votre contrat de support, ou envoyez un courriel à [emergency@vaultwares.com](mailto:emergency@vaultwares.com) avec **[P1]** dans l'objet.
  </Step>
</Steps>
