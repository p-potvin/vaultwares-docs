## Vue d'ensemble

Une maintenance régulière maintient votre déploiement VaultWares sécurisé, performant et conforme. Ce guide couvre les mises à jour de micrologiciel, les mises à jour logicielles, la rotation des clés, les tâches de maintenance matérielle et la vérification des sauvegardes.

<Note>
  Planifiez les activités de maintenance pendant les fenêtres de changement approuvées. Pour les déploiements à haute disponibilité, de nombreuses tâches peuvent être effectuées sans temps d'arrêt en utilisant des procédures de mise à jour progressive (rolling update).
</Note>

## Mises à jour de micrologiciel

### Vérifier les mises à jour disponibles

```bash
# Check all managed hardware for available firmware updates
vw-admin updates check \
  --scope firmware \
  --include-beta false \
  --format table

# Output:
# Device               Current     Available   Channel   Severity
# VaultDrive Fleet     3.1.2       3.2.0       Stable    Security
# VaultHSM PCIe-01     5.4.1       5.5.0       Stable    Feature
# VaultScan-01         2.3.0       2.3.1       Stable    Bugfix
# VaultGate-01         4.2.3       4.3.0       Stable    Security
```

### Mise à jour du micrologiciel VaultDrive

```bash
# Update a single device
vw-storage firmware-update \
  --device /dev/sdb \
  --version latest \
  --verify-signature true

# Update an entire fleet via Management Console
vw-admin fleet-update firmware \
  --device-group AllVaultDrives \
  --version 3.2.0 \
  --schedule "2024-11-20T02:00:00" \
  --batch-size 50 \
  --rollback-on-failure true

# Monitor fleet update progress
vw-admin fleet-update status --job job_abc123
```

### Mise à jour du micrologiciel VaultHSM

<Warning>
  Les mises à jour du micrologiciel HSM sont des opérations critiques. Sauvegardez toujours toutes les clés avant de mettre à jour. Pour les HSM réseau dans un cluster, mettez à jour un nœud à la fois.
</Warning>

<Steps>
  <Step title="Sauvegarder toutes les clés">
    ```bash
    vw-hsm backup create \
      --all-keys \
      --output /secure/backup/hsm-backup-$(date +%Y%m%d).enc \
      --split 3-of-5 \
      --custodians alice,bob,charlie,diana,erin
    ```
  </Step>
  <Step title="Vérifier la sauvegarde">
    ```bash
    vw-hsm backup verify \
      --input /secure/backup/hsm-backup-$(date +%Y%m%d).enc \
      --custodian-keys alice.key,bob.key,charlie.key
    ```
  </Step>
  <Step title="Télécharger et vérifier le paquet de micrologiciel">
    ```bash
    # Download firmware
    curl -O https://updates.vaultwares.com/hsm/firmware/vaulthsm-5.5.0.pkg

    # Verify signature
    vw-hsm firmware verify \
      --package vaulthsm-5.5.0.pkg \
      --signature vaulthsm-5.5.0.pkg.sig \
      --public-key /etc/vaultwares/firmware-signing.pub
    ```
  </Step>
  <Step title="Mettre à jour (cluster : un nœud à la fois)">
    ```bash
    # Remove node from cluster before update
    vw-hsm cluster drain --node hsm01 --graceful

    # Apply firmware update
    vw-hsm firmware-update \
      --host hsm01.corp.example.com \
      --package vaulthsm-5.5.0.pkg \
      --confirm

    # Wait for reboot and verify
    sleep 120
    vw-hsm status --host hsm01.corp.example.com

    # Re-add to cluster
    vw-hsm cluster add-member --node hsm01
    vw-hsm cluster sync --force
    ```
  </Step>
</Steps>

### Mise à jour du micrologiciel VaultGate

```bash
# VaultGate supports hot-swap upgrade with zero downtime (HA pairs)
# Upgrade the standby unit first
vaultgate ha failover --to vaultgate-02 --graceful

# Update the now-standby vaultgate-01
vaultgate firmware-update \
  --target vaultgate-01 \
  --version 4.3.0 \
  --verify-signature

# Fail back after update completes
vaultgate ha failback --graceful
```

## Mises à jour logicielles

### Canaux de mise à jour

| Canal | Description | Recommandé pour |
|---------|-------------|-----------------|
| **Stable** | Versions rigoureusement testées | Tous les environnements de production |
| **LTS** | Support à long terme avec correctifs de sécurité étendus | Environnements réglementés, fenêtres de changement minimales |
| **Beta** | Tests de pré-publication | Environnements non de production uniquement |

```bash
# Configure update channel
vw-admin updates set-channel \
  --channel stable \
  --auto-download true \
  --auto-install false \
  --notify-email security@example.com

# View available software updates
vw-admin updates check --scope software --format table
```

### Mise à jour de VaultCrypt

```bash
# Linux (systemd)
sudo apt-get update && sudo apt-get install vaultcrypt-enterprise
sudo systemctl restart vaultcrypt

# Windows (PowerShell — requires admin)
Invoke-WebRequest "https://releases.vaultwares.com/vaultcrypt/3.2.1/vaultcrypt-3.2.1-x64.msi" -OutFile vaultcrypt-update.msi
Start-Process msiexec -ArgumentList "/i vaultcrypt-update.msi /quiet /norestart" -Wait

# Verify version after update
vaultcrypt --version
```

### Mise à jour de VaultAccess (Docker)

```bash
# Pull new image
docker pull registry.vaultwares.com/vaultaccess:3.2.1

# Perform rolling update with docker compose
docker compose pull vaultaccess
docker compose up -d --no-deps vaultaccess

# Or with Kubernetes (zero-downtime rolling update)
kubectl set image deployment/vaultaccess \
  vaultaccess=registry.vaultwares.com/vaultaccess:3.2.1 \
  -n vaultwares

kubectl rollout status deployment/vaultaccess -n vaultwares
```

## Calendrier de rotation des clés

### Rotation automatique des clés

```bash
# Configure automated rotation for TLS keys
vw-admin key-rotation schedule \
  --key-pattern "TLS-*" \
  --interval 365d \
  --notification-lead 30d \
  --auto-rotate true \
  --alert-email security@example.com

# Configure rotation for data encryption keys
vw-admin key-rotation schedule \
  --key-pattern "DEK-*" \
  --interval 365d \
  --notification-lead 45d \
  --auto-rotate false \
  --require-approval-from security-officer@example.com
```

### Procédure de rotation manuelle des clés

```bash
# 1. Generate new key version
vw-hsm generate-key \
  --type aes \
  --size 256 \
  --label "DB-Master-Key-v2" \
  --extractable false

# 2. Re-wrap DEKs with the new master key
vw-hsm key-rewrap \
  --old-kek "DB-Master-Key-v1" \
  --new-kek "DB-Master-Key-v2" \
  --target-deks "DB-PII-DEK-v1,DB-Financial-DEK-v1"

# 3. Update application configuration to reference new key label
vw-admin config update \
  --key master_key_label \
  --value "DB-Master-Key-v2"

# 4. Verify applications are using the new key
vw-admin key-rotation verify --key "DB-Master-Key-v2"

# 5. Schedule deprecation of old key (keep for decryption of old backups)
vw-hsm key deprecate \
  --label "DB-Master-Key-v1" \
  --deprecation-date "2025-11-01"
```

## Maintenance matérielle

### Liste de contrôle d'inspection régulière

<AccordionGroup>
  <Accordion title="Inspection matérielle mensuelle">
    - [ ] Vérifier les LED de statut sur tous les appareils VaultHSM (toutes devraient être vertes)
    - [ ] Vérifier l'état de l'UPS ou de l'alimentation double (PSU) via la console de gestion
    - [ ] Inspecter les connexions des câbles — vérifier les câbles desserrés ou endommagés
    - [ ] Nettoyer l'équipement de rack avec de l'air comprimé (éviter les liquides près de l'électronique)
    - [ ] Vérifier les journaux de température et d'humidité de la salle de serveurs
    - [ ] Vérifier que les scellés anti-effraction sur toutes les unités VaultHSM sont intacts
  </Accordion>
  <Accordion title="Maintenance de la batterie / du condensateur VaultHSM">
    Les unités VaultHSM utilisent un supercondensateur pour préserver les clés résidentes en RAM pendant de brèves interruptions de courant. Le condensateur doit être testé annuellement.

    ```bash
    # Test supercapacitor health
    vw-hsm hardware-test --component supercap --node hsm01

    # Output:
    # Supercapacitor status: HEALTHY
    # Estimated hold-up time: 42 seconds (nominal: 30-60s)
    # Last test: 2024-06-01
    ```
  </Accordion>
  <Accordion title="Ventilateur et température">
    ```bash
    # Check HSM appliance temperature and fan speed
    vw-hsm hardware-status --node hsm01 --component thermal

    # Output:
    # CPU temperature:  42°C (max: 70°C)
    # Board temperature: 38°C (max: 60°C)
    # Fan 1 speed:      3,200 RPM (status: Normal)
    # Fan 2 speed:      3,150 RPM (status: Normal)
    ```
  </Accordion>
</AccordionGroup>

## Vérification des sauvegardes

```bash
# Monthly backup verification
vw-backup verify-all \
  --repo backup://nas01/vaults/prod \
  --decrypt-key hsm:DB-Backup-Key \
  --threads 4 \
  --output-report /reports/backup-verify-$(date +%Y%m).json

# Quarterly restore test
vw-backup restore \
  --repo backup://nas01/vaults/prod \
  --recovery-point "$(vw-backup recovery-points list --latest)" \
  --destination /data/restore-test-$(date +%Y%m) \
  --decrypt-key hsm:DB-Backup-Key

# Verify restore completeness
vw-backup verify-restore \
  --source /data/restore-test-$(date +%Y%m) \
  --compare-to-backup \
  --format report \
  --output /reports/restore-test-$(date +%Y%m).html
```

## Procédures de fenêtre de maintenance

### Liste de contrôle avant la maintenance

```bash
# Pre-maintenance health snapshot
vw-admin health-check --full --output /reports/pre-maint-$(date +%Y%m%d).json

# Notify monitoring system
vw-monitor maintenance-mode enable \
  --duration 2h \
  --suppress-alerts \
  --reason "Scheduled firmware updates"

# Verify backups are current
vw-backup jobs list --status success --since 24h
```

### Validation après la maintenance

```bash
# Post-maintenance health check
vw-admin health-check --full --output /reports/post-maint-$(date +%Y%m%d).json

# Compare pre- and post-maintenance configs
vw-admin config diff \
  --before /reports/pre-maint-$(date +%Y%m%d).json \
  --after /reports/post-maint-$(date +%Y%m%d).json

# Restore normal monitoring
vw-monitor maintenance-mode disable

# Verify all services are healthy
vw-admin service status --all
```
