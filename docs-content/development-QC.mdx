<Info>
  **Prérequis** :
  - Node.js 20+ (Node 22 recommandé)
  - Pages sources `docs-content/` avec frontmatter YAML
</Info>

## Pourquoi cette page existe?

Si tu veux **modifier la documentation**, tu veux pouvoir :

1) faire un changement,
2) le voir sur ton écran,
3) l’envoyer (PR) pour révision.

Cette page t’explique ça sans détour.

<Steps>
<Step title="Installer les dépendances">

```bash
npm ci
```
</Step>

<Step title="Générer les ressources de pages">

L'interface consomme des ressources de langue générées sous `src/resources/`. Régénérez-les après avoir modifié `docs-content/` :

```bash
npm run generate:page-resources
```

</Step>
</Steps>

## Aperçu local

Démarre le serveur local (ça te donne un lien à ouvrir dans ton navigateur) :

```bash
npm run dev
```

Vite utilise le port 5173 par défaut et choisit le prochain port disponible s'il est occupé. Vous pouvez forcer un port :

```bash
npm run dev -- --port 3333
```

## Build et aperçu

Build :

```bash
npm run build
```

Prévisualiser le build de production localement :

```bash
npm run preview
```

## Déploiement

### Important : GitHub ne déploie pas

On n’utilise pas GitHub Actions pour déployer en prod.

Le modèle canonique est documenté ici :

- `/operations/deployment-flow` (version QC)

Si tu es juste en train d’éditer des docs, ton “déploiement” à toi, c’est : **voir les changements en local** + ouvrir une PR.

<Tip>
  Tu peux ignorer la partie “infra” si tu fais juste des corrections de texte.
</Tip>

## Formatage du code

Nous suggérons d'utiliser des extensions dans votre IDE pour reconnaître et formater le MDX. Si vous êtes utilisateur de VSCode, envisagez l'[extension MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique et [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour le formatage du code.

## Dépannage

<AccordionGroup>
  <Accordion title='Problème : version de Node incompatible'>

    Il s'agit généralement d'une version de Node incompatible. Utilisez Node 20+ (Node 22 recommandé) et réinstallez :
    1. Supprimez `node_modules/` et le lockfile seulement si nécessaire
    2. Exécutez `npm ci`
  </Accordion>

  <Accordion title="Problème : le contenu semble périmé après modification de docs-content/">

    Solution : régénérez les ressources et redémarrez le serveur :
    1. `npm run generate:page-resources`
    2. Redémarrez `npm run dev`
  </Accordion>
</AccordionGroup>
