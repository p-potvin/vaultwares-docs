<Info>
  **Prérequis** :
  - Node.js version 19 ou supérieure
  - Un dépôt de documentation avec un fichier `docs.json`
</Info>

Suivez ces étapes pour installer et exécuter VaultWares Docs sur votre système d'exploitation.

<Steps>
<Step title="Installer l'interface de ligne de commande (CLI) VaultWares Docs">

```bash
npm i -g mint
```
</Step>

<Step title="Aperçu local">

Naviguez jusqu'au répertoire de votre documentation où se trouve votre fichier `docs.json`, puis exécutez la commande suivante :

```bash
mint dev
```

Un aperçu local de votre documentation sera disponible à `http://localhost:3000`.

</Step>
</Steps>

## Ports personnalisés

Par défaut, VaultWares Docs utilise le port 3000. Vous pouvez personnaliser le port sur lequel VaultWares Docs s'exécute en utilisant le drapeau `--port`. Par exemple, pour exécuter VaultWares Docs sur le port 3333, utilisez cette commande :

```bash
mint dev --port 3333
```

Si vous tentez d'exécuter VaultWares Docs sur un port déjà utilisé, il utilisera le prochain port disponible :

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Versions de VaultWares Docs

Veuillez noter que chaque version de l'interface de ligne de commande (CLI) est associée à une version spécifique de VaultWares Docs. Si votre aperçu local ne correspond pas à la version de production, veuillez mettre à jour l'interface de ligne de commande (CLI) :

```bash
npm mint update
```

## Validation des liens

L'interface de ligne de commande (CLI) peut aider à valider les liens de votre documentation. Pour identifier tout lien brisé, utilisez la commande suivante :

```bash
mint broken-links
```

Nous suggérons d'utiliser des extensions dans votre IDE pour reconnaître et formater le MDX. Si vous êtes utilisateur de VSCode, envisagez l'[extension MDX VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique et [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour le formatage du code.

## Dépannage

<AccordionGroup>
  <Accordion title='Error: Could not load the "sharp" module using the darwin-arm64 runtime'>

    Ceci peut être dû à une version obsolète de Node. Essayez ce qui suit :
    1. Supprimer la version actuelle de l'interface de ligne de commande (CLI) : `npm remove -g mint`
    2. Mettre à niveau vers Node v19 ou supérieur.
    3. Réinstaller l'interface de ligne de commande (CLI) : `npm i -g mint`
  </Accordion>

  <Accordion title="Issue: Encountering an unknown error">

    Solution : Allez à la racine de votre appareil et supprimez le dossier `~/.VaultWares Docs`. Ensuite, exécutez à nouveau `mint dev`.
  </Accordion>
</AccordionGroup>

Curieux de savoir ce qui a changé dans la dernière version de l'interface de ligne de commande (CLI) ? Consultez le [journal des modifications de l'interface de ligne de commande (CLI)](https://www.npmjs.com/package/VaultWares Docs?activeTab=versions).
