# Vérification

- ID de catégorie : `VERIFICATION`
- Source du résumé : `instructions/summaries/VERIFICATION.md`
- Source des notes : `instructions/notes/VERIFICATION.md`

S'applique quand : on affirme que quelque chose fonctionne ou est terminé.
À faire :
- Préférer la vérification la plus simple qui prouve le changement.
- Enregistrer ce qui a été réellement exécuté/vérifié (commandes, pages visitées, résultats).
À ne pas faire :
- Affirmer que ça « fonctionne » juste à cause d'un seul 200 OK.
Terminé quand :
- Les preuves de vérification sont enregistrées et reproductibles.

## Notes
Les assistants lisent les notes complètes seulement quand on les y invite explicitement : `read full notes`.
Chemin des notes : `instructions/notes/VERIFICATION.md`
