# Refactorisation du nettoyage

- ID de catégorie : `CLEANUP_REFACTOR`
- Source du résumé : `instructions/summaries/CLEANUP_REFACTOR.md`
- Source des notes : `instructions/notes/CLEANUP_REFACTOR.md`

S'applique dans les cas suivants : refactorisation, désencombrement, réorganisation, modifications importantes sans nouvelle fonctionnalité destinée à l'utilisateur.
Faire :
- Rédigez un plan de nettoyage avant de le modifier.
- Verrouillez d'abord le comportement avec des tests ou des étapes de vérification concrètes.
- Préférer la suppression à l'ajout ; garder les différences petites et réversibles.
Pas :
- Combiner le refactor avec des travaux de fonctionnalités sans rapport.
Fais quand :
- Le comportement est inchangé (ou explicitement déclaré) et des preuves de vérification existent.

## Remarques
Les assistants ne lisent les notes complètes que lorsqu'ils y sont explicitement invités : « lire les notes complètes ».
Chemin des notes : `instructions/notes/CLEANUP_REFACTOR.md`
