# Nettoyage et Refactorisation

- ID de catégorie : `CLEANUP_REFACTOR`
- Source du résumé : `instructions/summaries/CLEANUP_REFACTOR.md`
- Source des notes : `instructions/notes/CLEANUP_REFACTOR.md`

S'applique quand : refactorisation, désencombrement, réorganisation, grandes modifications sans nouvelle fonctionnalité visible par l'utilisateur.
À faire :
- Écrire un plan de nettoyage avant de modifier.
- Verrouiller le comportement avec des tests ou des étapes de vérification concrètes en premier.
- Préférer la suppression plutôt que l'ajout; garder les différences (diffs) petites et réversibles.
À ne pas faire :
- Mélanger la refactorisation avec des fonctionnalités sans rapport.
Terminé quand :
- Le comportement n'a pas changé (ou est explicitement déclaré) et des preuves de vérification existent.

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on le leur demande explicitement : `read full notes`.
Chemin des notes : `instructions/notes/CLEANUP_REFACTOR.md`
