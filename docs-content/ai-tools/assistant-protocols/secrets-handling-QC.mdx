# Gestion des secrets

- ID de catégorie : `SECRETS_HANDLING`
- Source du résumé : `instructions/summaries/SECRETS_HANDLING.md`
- Source des notes : `instructions/notes/SECRETS_HANDLING.md`

S'applique aux cas où : .env, tokens, identifiants, clés SSH, clés API, VaultWarden, systèmes de secrets.
À faire :
- Ne pas enregistrer les secrets dans le registre de l'agent, les documents, les problèmes ou les transcriptions de terminal.
- Préférer stocker les secrets dans VaultWarden (`warden.vaultwares.ca`) maintenant.
- Faire référence aux noms/emplacements des secrets, pas aux valeurs.
À ne pas faire :
- Commiter les secrets ou les coller dans du markdown.
C'est fait quand :
- Aucun secret n'apparaît dans les diffs, les journaux ou les réponses.

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on le leur demande explicitement : `read full notes`.
Chemin des notes : `instructions/notes/SECRETS_HANDLING.md`
