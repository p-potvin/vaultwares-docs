# Normes de documentation

- ID de catégorie : `DOCS_STANDARDS`
- Source de résumé : `instructions/summaries/DOCS_STANDARDS.md`
- Source de notes : `instructions/notes/DOCS_STANDARDS.md`

S'applique quand : on ajoute ou modifie des pages de contenu de documentation ou la navigation de documentation.
À faire :
- Chaque page doit avoir un frontmatter YAML avec un titre et une description.
- Utiliser la deuxième personne du singulier ; des liens internes relatifs ; des balises de langue sur les blocs de code.
- Ajouter les nouvelles pages à la navigation docs.json.
À ne pas faire :
- Créer des pages orphelines qui ne sont pas dans la navigation.
Terminé quand :
- La page est rendue, l'entrée de navigation existe, et la vérification est enregistrée.

## Notes
Les assistants lisent les notes complètes seulement quand on le leur demande explicitement : `read full notes`.
Chemin des notes : `instructions/notes/DOCS_STANDARDS.md`
