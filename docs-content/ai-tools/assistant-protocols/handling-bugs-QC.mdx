# Gestion des bogues

- ID de catégorie : `HANDLING_BUGS`
- Source du résumé : `instructions/summaries/HANDLING_BUGS.md`
- Source des notes : `instructions/notes/HANDLING_BUGS.md`

S'applique quand : un bogue/défaut est signalé ou découvert.
À faire :
- Créer un problème GitHub d'abord (titre, reproduction, attendu vs réel, environnement).
- Décider de corriger immédiatement ou d'assigner en utilisant le budget de jetons : corriger immédiatement seulement si le cycle complet (diagnostic+patch+vérification+réponse) est censé tenir dans <=6000 jetons.
- Si ce n'est pas une correction immédiate : assigner le problème à l'utilisateur et y faire référence dans la réponse.
À ne pas faire :
- Commencer un travail de correction de bogue important sans lien de problème.
Terminé quand :
- Le problème existe et est lié ; la correction est soit fusionnée via une PR, soit clairement reportée avec un responsable.

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on le leur demande explicitement : `read full notes`.
Chemin des notes : `instructions/notes/HANDLING_BUGS.md`
