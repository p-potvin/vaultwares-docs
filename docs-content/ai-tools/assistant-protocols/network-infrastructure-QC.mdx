# Infrastructure réseau

- Category ID: `NETWORK_INFRASTRUCTURE`
- Summary source: `instructions/summaries/NETWORK_INFRASTRUCTURE.md`
- Notes source: `instructions/notes/NETWORK_INFRASTRUCTURE.md`

S'applique quand : tailscale/tailnet, SSH, VPS, ports, nginx, pare-feu, exécuteurs CI, tout accès réseau.
À faire :
- Tailnet est le plan de contrôle par défaut. Les services privés sont uniquement tailnet.
- Le SSH vers les serveurs est uniquement tailnet. Préférer SSH plutôt que les adresses IP Tailscale ou tailnet.
- Pas de contact externe pour les opérations sensibles : ne pas ouvrir de ports entrants pour l'accès administratif ; ne pas ajouter de dépendances de sortie publiques sans approbation explicite.
- Pour les boucles ou les lots de requêtes TCP/UDP/HTTP/API, appliquer REQUEST_RATE_LIMITING avant d'exécuter des commandes.
- CI : ne pas dépendre des exécuteurs GitHub hébergés à l'extérieur pour l'infrastructure VaultWares ; utiliser uniquement des exécuteurs auto-hébergés.
- Consulter ces pages SoT avant les changements :
  - vaultwares-docs/docs-content/operations/network-map.mdx
  - vaultwares-docs/docs-content/operations/tailscale.mdx
Ne pas :
- Ajouter des chemins d'accès publics « rapides et temporaires ».
Terminé quand :
- Les chemins d'accès sont prioritaires tailnet et documentés ; la vérification comprend la connectivité depuis la perspective tailnet prévue.
