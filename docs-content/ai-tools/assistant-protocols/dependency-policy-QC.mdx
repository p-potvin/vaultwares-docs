# Politique de dépendances

- ID de catégorie : `DEPENDENCY_POLICY`
- Source de résumé : `instructions/summaries/DEPENDENCY_POLICY.md`
- Source de notes : `instructions/notes/DEPENDENCY_POLICY.md`

S'applique lors de : l'ajout ou la mise à jour de dépendances, de chaînes d'outils (toolchains), de fichiers de verrouillage (lockfiles), de gestionnaires de paquets.
À faire :
- Ne pas ajouter de nouvelles dépendances sauf si explicitement demandé ou clairement nécessaire et justifié.
- Préférer les bibliothèques/utilitaires existants déjà utilisés dans le dépôt.
- Enregistrer les changements de dépendances et pourquoi.
À ne pas faire :
- Introduire de nouveaux outils « juste parce que ».
Terminé lorsque :
- Le changement de dépendance est minimal, justifié et vérifié (build/tests).

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on le leur demande explicitement : `read full notes`.
Chemin des notes : `instructions/notes/DEPENDENCY_POLICY.md`
