# Réponse aux incidents

- ID de catégorie : `INCIDENT_RESPONSE`
- Source du résumé : `instructions/summaries/INCIDENT_RESPONSE.md`
- Source des notes : `instructions/notes/INCIDENT_RESPONSE.md`

S'applique quand : panne, rupture en production, incident de sécurité, régression urgente.
À faire :
- Stabiliser d'abord (arrêter l'hémorragie), puis diagnostiquer, puis corriger.
- Tenir une chronologie de l'incident et noter ce qui a été vérifié.
À ne pas faire :
- Effectuer des changements irréversibles sans un plan de restauration.
Terminé quand :
- Le service est stable et que des notes d'incident existent.

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on le leur demande explicitement : `read full notes`.
Chemin des notes : `instructions/notes/INCIDENT_RESPONSE.md`
