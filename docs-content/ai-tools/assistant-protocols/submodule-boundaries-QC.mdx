# Limites de sous-modules

- Category ID: `SUBMODULE_BOUNDARIES`
- Summary source: `instructions/summaries/SUBMODULE_BOUNDARIES.md`
- Notes source: `instructions/notes/SUBMODULE_BOUNDARIES.md`

S'applique lorsque : le chemin que vous modifiez est un sous-module git ou une copie de fournisseur miroir.
À faire :
- Si le changement cible `vaultwares-themes`, `vaultwares-adk`, ou `agent-ledger`, préférez modifier le dépôt autonome sous C:\\Users\\Administrator\\Desktop\\Github Repos\\<repo> (pas un *checkout* de sous-module).
- Si vous devez lire une copie de sous-module pour comprendre le comportement, traitez-la comme étant en lecture seule.
À ne pas faire :
- Appliquer de correctifs aux dépôts de Niveau 2 à l'intérieur du répertoire de sous-module d'un autre dépôt.
Terminé lorsque :
- Le correctif final est appliqué dans le *checkout* du dépôt autonome.

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on leur demande explicitement : `read full notes`.
