# Politique de déploiement

- ID de catégorie : `DEPLOYMENT_POLICY`
- Source du résumé : `instructions/summaries/DEPLOYMENT_POLICY.md`
- Source des notes : `instructions/notes/DEPLOYMENT_POLICY.md`

S'applique quand : déploiement, changements de VPS, opérations de publication, routage de production.
À faire :
- Administrateur Tailnet-d'abord ; minimiser le rayon d'impact ; conserver le chemin de retour arrière.
- Vérifier le comportement de déploiement avec des sondes concrètes et des vérifications d'interface utilisateur lorsque c'est pertinent.
À ne pas faire :
- Déployer à partir d'hypothèses ; vérifier l'état réel de l'environnement.
Terminé quand :
- Les étapes de déploiement et les preuves de vérification sont enregistrées.

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on le leur demande explicitement : `lire les notes complètes`.
Chemin des notes : `instructions/notes/DEPLOYMENT_POLICY.md`
