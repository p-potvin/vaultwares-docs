# Politique des branches Git

- ID de catégorie : `GIT_BRANCH_POLICY`
- Source du résumé : `instructions/summaries/GIT_BRANCH_POLICY.md`
- Source des notes : `instructions/notes/GIT_BRANCH_POLICY.md`

S'applique quand : toute action de flux de travail Git, les commits, les branches, les pushes.
À faire :
- Si sur la branche `main` : ne pas commiter à moins que l'utilisateur n'ait explicitement demandé des commits dans l'invite actuelle.
- Préférer une branche de fonctionnalité (préfixe par défaut : vw-codex-).
- Garder les changements non liés hors de la branche.
À ne pas faire :
- Commiter/pousser sur `main` sans instruction explicite.
Terminé quand :
- L'état de la branche est propre, isolé et prêt pour examen.

## Notes
Les assistants lisent les notes complètes seulement lorsqu'on le leur demande explicitement : `read full notes`.
