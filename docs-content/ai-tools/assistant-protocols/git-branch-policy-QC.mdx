# Politique de branche Git

- ID de catégorie : `GIT_BRANCH_POLICY`
- Source du résumé : `instructions/summaries/GIT_BRANCH_POLICY.md`
- Source des notes : `instructions/notes/GIT_BRANCH_POLICY.md`

S'applique lorsque : toute action de workflow git, validations, branches, push.
Faire :
- Si sur la branche principale : ne validez pas à moins que l'utilisateur n'ait explicitement demandé une validation dans l'invite actuelle.
- Préférer une branche de fonctionnalité (préfixe par défaut : vw-codex-).
- Gardez les changements non reliés hors de la branche.
Pas :
- Engager/pousser sur main sans instruction explicite.
Fais quand :
- L'état de la branche est propre, isolé et prêt à être examiné.

## Remarques
Les assistants ne lisent les notes complètes que lorsqu'ils y sont explicitement invités : « lire les notes complètes ».
Chemin des notes : `instructions/notes/GIT_BRANCH_POLICY.md`
