# Pr Policy

- Category ID: `PR_POLICY`
- Summary source: `instructions/summaries/PR_POLICY.md`
- Notes source: `instructions/notes/PR_POLICY.md`

Applies when: creating or updating PRs, requesting review, CI enforcement.
Do:
- Keep PR scope tight; include verification steps and evidence.
- Add the required reviewers specified by repo policy.
Do not:
- Merge directly to main unless explicitly instructed.
Done when:
- PR is published and includes what changed, how verified, and remaining risks.
