# Posture de sécurité

- ID de catégorie : `SECURITY_POSTURE`
- Source du résumé : `instructions/summaries/SECURITY_POSTURE.md`
- Source des notes : `instructions/notes/SECURITY_POSTURE.md`

S'applique aux cas où : auth, crypto, keys, tokens, contrôle d'accès, modèle de menace, expérience utilisateur de sécurité (UX).
À faire :
- La confidentialité d'abord. La sécurité au service de la confidentialité.
- Utiliser la cryptographie post-quantique avec ML-KEM lorsque l'UX de sécurité ou les protocoles exigent la PQC.
- S'assurer que les serveurs ne lisent jamais, ne conservent jamais et ne reconstruisent jamais les clés privées ou les clés de déchiffrement partagées.
À ne pas faire :
- Ajouter des fonctionnalités qui impliquent l'usurpation d'identité, le contournement de KYC ou la falsification de documents.
Fait lorsque :
- Vous pouvez expliquer comment le changement préserve les contraintes de connaissance nulle/séparation des clés.
