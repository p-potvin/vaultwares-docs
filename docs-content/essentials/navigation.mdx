VaultWares Docs does not use a `docs.json` navigation file. The sidebar is generated from your `docs-content/` pages.

## Where the sidebar comes from

1. Source pages live under `docs-content/**/*.mdx` (EN), optionally paired with `*-QC.mdx` (QC).
2. Run `npm run generate:page-resources` to generate:
   - `src/resources/pages/*.json` (EN/QC bodies)
   - `src/resources/pageResourcesManifest.ts` (route index)
3. The app uses the generated manifest to build the sidebar and routes.

## Route rules

- `docs-content/index.mdx` → `/`
- `docs-content/getting-started/overview.mdx` → `/getting-started/overview`
- `docs-content/hardware/index.mdx` → `/hardware`
- `*-QC.mdx` is the QC locale for the same route (the toggle switches languages; QC falls back to EN when missing).

<Warning>
Avoid creating pages whose top-level route conflicts with static assets in `public/` (for example `/images`) or other reserved paths. Prefer descriptive top-level folders like `api-reference`, `getting-started`, `operations`, etc.
</Warning>

## Ordering and labels

- **Section groups** are derived from the first route segment (`/hardware/...` groups under `hardware`).
- Group order is controlled by `SECTION_ORDER` in `src/resources/uiResources.ts`.
- Page labels come from each page frontmatter `title` (if missing, the route slug is used as a fallback).
- Within a group, routes are currently sorted by `routePath`.

## Add a new page

1. Create `docs-content/<section>/<page>.mdx` and optionally `<page>-QC.mdx`.
2. Ensure frontmatter includes `title` and `description`.
3. Run `npm run generate:page-resources` and restart the dev server if needed.
