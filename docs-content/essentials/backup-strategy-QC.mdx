VaultWares utilise le protocole **A.N.S.W.E.R.S.** (Alternating, Non Sequential, With Encryption, Restoring System) pour assurer une durabilité maximale des données.

## 1. Cibles

Les sauvegardes sont synchronisées sur les cibles suivantes :
- Machine locale
- Greencloud VPS
- Ancien VPS
- Google Drive
- Vaultwarden auto-hébergé
- Proton Drive
- iCloud
- Mega

## 2. Obligation de chiffrement

> [!CAUTION]
> **Aucune donnée ne doit quitter l'environnement local sans être chiffrée.**
> Toutes les sauvegardes multi-sites doivent être chiffrées en transit vers un VPS ou des cibles cloud.

## 3. Logique de rotation (A.N.S.W.E.R.S.)

Pour prolonger la période de couverture tout en conservant les sauvegardes récentes, utilisez la logique de rotation mathématique suivante ($N=20$) :

1. **Calculer la valeur de saut** : `skip_value = today_count + skip_modifier` (où `skip_modifier` est par défaut de 1).
2. **Sélectionner la cible à supprimer** :
   - Si `today_count` est **impair** : Supprimer l'enregistrement à l'index `N - skip_value`.
   - Si `today_count` est **pair** : Supprimer l'enregistrement à l'index `0 + skip_value`.
3. **Mettre à jour l'état** : Incrémenter `skip_modifier`. Réinitialiser à 1 une fois que `skip_value >= (N/2) - 1`.

## 4. Règle d'automatisation

Le script `sync-global-instructions.ps1` et la routine de synchronisation de sauvegarde ne doivent **jamais** être exécutés automatiquement. Ils exigent une conversation active et une confirmation explicite de l'utilisateur.
