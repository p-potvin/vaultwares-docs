VaultWares Docs n'utilise pas de fichier de navigation `docs.json`. La barre latérale est générée à partir des pages `docs-content/`.

## D'où vient la barre latérale

1. Les pages sources vivent sous `docs-content/**/*.mdx` (EN), et peuvent être jumelées à `*-QC.mdx` (QC).
2. Exécutez `npm run generate:page-resources` pour générer :
   - `src/resources/pages/*.json` (corps EN/QC)
   - `src/resources/pageResourcesManifest.ts` (index des routes)
3. L'application utilise le manifeste généré pour construire la barre latérale et les routes.

## Règles de routes

- `docs-content/index.mdx` → `/`
- `docs-content/getting-started/overview.mdx` → `/getting-started/overview`
- `docs-content/hardware/index.mdx` → `/hardware`
- `*-QC.mdx` est la version QC pour la même route (le bouton bascule la langue ; QC revient à EN si la version QC est absente).

<Warning>
Évitez de créer des pages dont la route de premier niveau entre en conflit avec des assets statiques dans `public/` (par exemple `/images`) ou d'autres chemins réservés. Préférez des dossiers de premier niveau descriptifs comme `api-reference`, `getting-started`, `operations`, etc.
</Warning>

## Ordre et libellés

- Les **sections** sont dérivées du premier segment de route (`/hardware/...` se regroupe sous `hardware`).
- L'ordre des sections est contrôlé par `SECTION_ORDER` dans `src/resources/uiResources.ts`.
- Les libellés viennent du frontmatter `title` (sinon, repli sur le slug).
- À l'intérieur d'une section, les routes sont actuellement triées par `routePath`.

## Ajouter une nouvelle page

1. Créez `docs-content/<section>/<page>.mdx` et optionnellement `<page>-QC.mdx`.
2. Assurez-vous que le frontmatter inclut `title` et `description`.
3. Exécutez `npm run generate:page-resources` et redémarrez le serveur de dev si nécessaire.
