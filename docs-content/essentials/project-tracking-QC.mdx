VaultWares utilise un système de suivi à niveaux pour maintenir la cohérence et la visibilité entre l'exécution à court terme et la vision à long terme.

## 1. Hiérarchie de la documentation

| File | Scope | Level |
| ---- | ----- | ----- |
| `TASKS.md` | Éléments d'exécution de course actifs | Atomique |
| `TODO.md` | Arriéré à court terme et prochaines étapes immédiates | Projet |
| `ROADMAP.md` | Référence à long terme et vision stratégique | Global |

## 2. Maintenance

Les agents sont responsables de la maintenance de ces fichiers pendant le Flux Multi-Agents :
- **Phase 1**: Lire `TODO.md` et `ROADMAP.md` pour établir la base de la nouvelle liste de tâches.
- **Phase 6**: Mettre à jour `TODO.md` avec les éléments complétés et identifier les nouvelles entrées d'arriéré.

## 3. Télémétrie

Toutes les données de suivi de projet sont complétées par le **système de Télémétrie des Agents**. Les statistiques de performance sont archivées toutes les **2 semaines** pour maintenir les fichiers de suivi actifs efficaces.
