VaultWares Docs est une application React + Vite personnalisée. Elle n'utilise pas de fichier de configuration `docs.json`.

Cette page indique où configurer le contenu, la navigation et le branding.

## Contenu et langues

- Les pages sources vivent dans `docs-content/**/*.mdx` (EN).
- Les pages QC sont optionnelles et se trouvent à côté des pages EN sous la forme `*-QC.mdx`.
- Les pages doivent inclure un frontmatter YAML (`title`, `description`).

Après avoir modifié `docs-content/`, régénérez les ressources de langue consommées par l'interface :

```bash
npm run generate:page-resources
```

## Navigation et libellés de sections

La navigation est générée à partir de `docs-content/` et du manifeste généré sous `src/resources/`.

- Le regroupement (sections de la barre latérale) vient du premier segment de route (`/hardware/...` → `hardware`).
- L'ordre des sections et les libellés bilingues vivent dans `src/resources/uiResources.ts` (`SECTION_ORDER` + `SECTION_LABELS`).
- Les libellés des pages viennent du frontmatter `title` (chargé par langue).

## Thème et jetons de marque

L'interface utilise les jetons VaultWares depuis `vaultwares-themes` :

- Les variables de jetons de base vivent dans `vaultwares-themes/vaultwares-revisited/revisited.css`.
- Le mapping Tailwind v4 et les ajustements typographiques vivent dans `src/index.css`.

Si vous avez besoin de nouveaux composants UI pour les pages (cards, callouts, tabs, accordéons), modifiez :

- `src/markdownComponents.tsx` (composants MDX + Markdown).

## Logo et favicon

- Le logo de la barre supérieure est configuré dans `src/App.tsx` et servi depuis `public/`.
- Le favicon est configuré dans `index.html` et servi depuis `public/`.
