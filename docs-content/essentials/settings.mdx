VaultWares Docs is a custom React + Vite app. It does not use a `docs.json` configuration file.

This page points to the places you actually configure content, navigation, and branding.

## Content and locales

- Source pages live in `docs-content/**/*.mdx` (EN).
- QC pages are optional and live alongside EN pages as `*-QC.mdx`.
- Pages must include YAML frontmatter (`title`, `description`).

After editing `docs-content/`, regenerate the locale resources consumed by the frontend:

```bash
npm run generate:page-resources
```

## Navigation and section labels

Navigation is generated from `docs-content/` and the generated manifest under `src/resources/`.

- Route grouping (sidebar sections) comes from the first route segment (`/hardware/...` → `hardware`).
- Section order and bilingual section labels live in `src/resources/uiResources.ts` (`SECTION_ORDER` + `SECTION_LABELS`).
- Page labels come from each page frontmatter `title` (loaded per locale).

## Theme and brand tokens

The docs UI uses the VaultWares design tokens from `vaultwares-themes`:

- Base token variables live in `vaultwares-themes/vaultwares-revisited/revisited.css`.
- Tailwind v4 theme mapping and typography overrides live in `src/index.css`.

If you need new UI patterns for docs pages (cards, callouts, tabs, accordions), update:

- `src/markdownComponents.tsx` (MDX + Markdown components).

## Logo and favicon

- Navbar logo is configured in `src/App.tsx` and served from `public/`.
- Favicon is configured in `index.html` and served from `public/`.
