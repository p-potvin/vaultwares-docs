## Aperçu

Ce guide couvre l'installation physique et la mise sous tension initiale de tous les produits matériels VaultWares. Suivez les instructions pour votre appareil spécifique. Des avertissements de sécurité et réglementaires sont inclus là où c'est applicable.

<Warning>
  Débranchez toujours votre système de l'alimentation secteur et éteignez-le avant d'installer des cartes PCIe ou d'effectuer des modifications physiques. Travaillez sur une surface antistatique et utilisez un bracelet antistatique mis à la terre lorsque vous manipulez des cartes de circuit imprimé.
</Warning>

## Stockage chiffré VaultDrive

Les appareils VaultDrive sont plug-and-play et ne nécessitent pas d'installation de pilote pour un fonctionnement de base.

<Steps>
  <Step title="Inspecter l'emballage">
    Vérifiez que le sceau anti-effraction de l'emballage est intact avant de l'ouvrir. Si le sceau est brisé, n'utilisez pas l'appareil — contactez immédiatement [support@vaultwares.com](mailto:support@vaultwares.com).
  </Step>
  <Step title="Connecter à l'hôte">
    Branchez le câble USB-C (inclus) sur un port USB 3.0 ou supérieur de votre ordinateur hôte. VaultDrive est alimenté par le bus — aucune alimentation externe n'est requise.

    | Couleur du LED | Statut |
    |-----------|--------|
    | Ambre clignotant | En attente d'authentification |
    | Vert fixe | Déverrouillé et monté |
    | Rouge clignotant | Échec d'authentification / verrouillage |
    | Bleu fixe | Mode lecture seule actif |
  </Step>
  <Step title="Exécuter l'utilitaire de configuration">
    L'appareil expose une petite partition lecture seule lors de la première connexion. Lancez l'utilitaire de configuration à partir de cette partition :

    ```bash
    # Linux
    /media/$USER/VAULTDRIVE/vaultdrive-setup

    # macOS
    open /Volumes/VAULTDRIVE/VaultWares-Setup.app
    ```
  </Step>
  <Step title="Définir le mot de passe maître et configurer les politiques">
    Suivez les instructions à l'écran pour définir un mot de passe maître. Pour les déploiements d'entreprise, importez un fichier de politique préconfiguré :

    ```bash
    vw-storage apply-policy \
      --device /dev/sdb \
      --policy /etc/vaultdrive/corporate-policy.json
    ```
  </Step>
  <Step title="Enregistrer dans la console de gestion VaultWares">
    ```bash
    vw-storage register \
      --device /dev/sdb \
      --console https://console.corp.example.com \
      --org-key ORG-KEY-XXXX
    ```
  </Step>
</Steps>

## VaultHSM PCIe

### Prérequis

- Slot PCIe x4 disponible (compatible avec x8 ou x16, physiquement x4)
- PCIe 3.0 ou supérieur recommandé pour un débit maximal
- Budget TDP disponible de 25W sur le slot PCIe
- Noyau Linux 4.18+ avec prise en charge du hotplug PCIe, ou Windows Server 2016+

### Installation

<Steps>
  <Step title="Préparer le serveur">
    Éteignez le serveur et déconnectez tous les câbles d'alimentation. Appuyez sur le bouton d'alimentation pour décharger les condensateurs résiduels. Ouvrez le châssis et localisez un slot PCIe x4 disponible.
  </Step>
  <Step title="Retirer le cache de slot">
    Retirez le cache de slot E/S correspondant du panneau arrière du châssis. Conservez la vis pour fixer la carte.
  </Step>
  <Step title="Insérer la carte">
    Retirez la carte VaultHSM PCIe de son sac antistatique. Alignez les contacts dorés avec le slot PCIe et appuyez fermement et uniformément jusqu'à ce que le loquet de rétention clique. Fixez le cache avec la vis conservée.

    <Note>
      La carte VaultHSM PCIe est livrée avec un cache de demi-hauteur préinstallé. Un cache de pleine hauteur est inclus dans la boîte pour les installations de châssis standard.
    </Note>
  </Step>
  <Step title="Réassembler et alimenter">
    Réattachez le cache du châssis, reconnectez les câbles d'alimentation et allumez le serveur.
  </Step>
  <Step title="Installer le pilote">
    ```bash
    # Linux
    sudo apt-get install vaulthsm-pcie-driver  # Debian/Ubuntu
    sudo dnf install vaulthsm-pcie-driver       # RHEL/Rocky/Alma

    # Verify the device is detected
    lspci | grep VaultHSM
    # Output: 0b:00.0 Encryption controller: VaultWares VaultHSM PCIe [1af4:1234]

    journalctl -u vaulthsm --no-pager | tail -20
    ```
  </Step>
  <Step title="Initialiser le HSM">
    ```bash
    vw-hsm init \
      --admin-pin <secure-admin-pin> \
      --so-pin <security-officer-pin> \
      --label "Production-PCIe-HSM-01"
    ```
  </Step>
</Steps>

## Appareil réseau VaultHSM

### Installation physique

<Steps>
  <Step title="Montage en rack">
    Le VaultHSM Network est un appareil de 1U. Montez-le dans un rack standard de 19 pouces à l'aide des oreilles de rack et des écrous de cage M6 fournis. Assurez-vous d'avoir au moins 1U d'espace libre au-dessus et en dessous pour un flux d'air adéquat.

    <Warning>
      Ne bloquez pas les fentes de ventilation avant ou arrière. L'appareil nécessite un flux d'air minimum de 100 LFM (pieds linéaires par minute) de l'avant vers l'arrière.
    </Warning>
  </Step>
  <Step title="Connecter l'alimentation">
    L'appareil est livré avec deux alimentations électriques redondantes. Connectez les deux câbles d'alimentation à des circuits ou des onduleurs séparés pour une disponibilité maximale. Les LED de statut de l'alimentation (panneau arrière) s'allumeront en vert lorsque l'alimentation est appliquée.
  </Step>
  <Step title="Connecter le réseau">
    Connectez les deux ports Gigabit Ethernet (LAN0, LAN1) à votre réseau de gestion. LAN0 est l'interface de gestion principale; LAN1 est utilisé pour le battement de cœur HA ou les connexions clients équilibrées en charge.
  </Step>
  <Step title="Configuration réseau initiale">
    Connectez un câble console série (RS-232, inclus) ou utilisez l'écran LCD du panneau avant pour définir l'adresse IP initiale :

    ```bash
    # Serial console: 9600 baud, 8-N-1
    screen /dev/ttyUSB0 9600

    # After login prompt (default: admin / VaultWares-Setup)
    configure network
    set ip 192.168.1.100/24
    set gateway 192.168.1.1
    set dns 8.8.8.8,8.8.4.4
    commit
    ```
  </Step>
  <Step title="Finaliser la configuration via CLI">
    ```bash
    # SSH to the appliance
    ssh admin@192.168.1.100

    # Initialize HSM
    vw-hsm init \
      --admin-pin <secure-admin-pin> \
      --so-pin <security-officer-pin> \
      --label "Production-Network-HSM-01"

    # Configure NTP (critical for audit log timestamps)
    vw-hsm ntp-config \
      --server ntp.corp.example.com \
      --backup-server pool.ntp.org
    ```
  </Step>
</Steps>

## Terminal biométrique VaultScan

### Montage

VaultScan peut être monté sur un bureau ou sur un mur à l'aide du matériel inclus.

<Steps>
  <Step title="Choisir l'emplacement de montage">
    Installez à une hauteur de 90 à 120 cm du sol pour un usage confortable. Assurez une ligne de visibilité dégagée d'au moins 30 cm devant le scanner. Évitez l'exposition directe au soleil pour le module de scan de l'iris.
  </Step>
  <Step title="Passer les câbles">
    VaultScan nécessite :
    - **Alimentation**: 12V CC 3A (compatible PoE+ — option câble unique)
    - **Réseau**: Gigabit Ethernet (pour la communication avec le serveur d'enrôlement)
    - **USB** (optionnel): Pour la connexion filaire au contrôleur d'accès de porte
    - **Wiegand / OSDP** (optionnel): Pour l'intégration avec les panneaux de contrôle d'accès existants
  </Step>
  <Step title="Monter la plaque arrière">
    Fixez la plaque arrière de montage au mur ou à la surface du bureau à l'aide des vis et des ancrages fournis. Faites passer les câbles par l'ouverture centrale.
  </Step>
  <Step title="Attacher le terminal">
    Clipsez le terminal VaultScan sur la plaque arrière montée. Connectez tous les câbles avant de clipser. Confirmez que le LED de statut passe au fixe ambre (en attente de configuration).
  </Step>
  <Step title="Configurer le terminal">
    ```bash
    # Discover VaultScan devices on the network
    vw-biometric discover --subnet 192.168.1.0/24

    # Configure a terminal
    vw-biometric configure \
      --device 192.168.1.51 \
      --name "ServerRoom-Entry-01" \
      --server https://vaultaccess.corp.example.com \
      --access-group ServerRoomAccess \
      --admin-pin <terminal-admin-pin>
    ```
  </Step>
  <Step title="Enrôler les utilisateurs">
    ```bash
    # Enroll a user's fingerprint + face
    vw-biometric enroll \
      --device 192.168.1.51 \
      --user alice@example.com \
      --modalities fingerprint,face \
      --fingers right-index,right-middle
    ```
  </Step>
</Steps>

## Appareil réseau VaultGate

### Installation physique

<Steps>
  <Step title="Montage en rack">
    VaultGate est un appareil réseau de 1U. Montez-le dans un rack standard de 19 pouces. Assurez un espace libre de 1U au-dessus et en dessous pour le flux d'air.
  </Step>
  <Step title="Connecter les interfaces">
    | Port | Objectif |
    |------|---------|
    | MGMT | Gestion hors bande (dédié) |
    | WAN0 / WAN1 | Liaison montante vers Internet ou pare-feu en amont |
    | LAN0–LAN7 | Segments réseau internes |
    | HA0 / HA1 | Battement de cœur haute disponibilité (déploiements double unité) |
  </Step>
  <Step title="Configuration initiale">
    Connectez-vous au port MGMT et accédez à l'assistant de configuration :

    ```bash
    # Default MGMT IP: 192.168.0.1/24
    ssh admin@192.168.0.1
    # Default password: VaultGate-Setup (change on first login)

    # Run initial setup wizard
    vaultgate setup-wizard
    ```
  </Step>
  <Step title="Activation de la licence">
    ```bash
    vaultgate license activate --key VWGT-XXXX-XXXX-XXXX-XXXX
    ```
  </Step>
  <Step title="Configurer les politiques de sécurité">
    ```bash
    # Import a pre-configured policy
    vaultgate policy import --file /etc/vaultgate/corporate-security-policy.xml

    # Apply and commit
    vaultgate policy apply --confirm
    ```
  </Step>
</Steps>

## Meilleures pratiques de câblage

<AccordionGroup>
  <Accordion title="Étiquetage des câbles">
    Étiquetez les deux extrémités de chaque câble au moment de l'installation. Utilisez un schéma de nommage cohérent : `[appareil]-[port]-[destination]`. Par exemple : `HSM01-LAN0-SW01-PORT12`.
  </Accordion>
  <Accordion title="Gestion des câbles">
    Utilisez des attaches en velcro (et non des serre-câbles) dans les environnements de rack pour faciliter les changements futurs. Faites passer les câbles d'alimentation séparément des câbles de données pour minimiser les interférences.
  </Accordion>
  <Accordion title="Chemins redondants">
    Pour les déploiements de production de HSM et VaultGate, connectez toujours les deux interfaces réseau à des commutateurs séparés (ou à des ports séparés sur un stack) pour une redondance physique.
  </Accordion>
</AccordionGroup>
