## Aperçu

Ce guide vous présente les étapes de configuration initiales qui doivent être complétées après l'installation de tout produit VaultWares. Suivez ces étapes dans l'ordre pour un déploiement fluide.

<Tip>
  Exécutez ces étapes d'abord dans votre environnement de *staging*. De nombreux paramètres (en particulier l'intégration de répertoire et l'initialisation du HSM) sont difficiles à modifier après que des données de production y aient été associées.
</Tip>

## Étape 1 : Configuration du compte administrateur

<Steps>
  <Step title="Se connecter avec les identifiants par défaut">
    Après l'installation, la console de gestion VaultWares est accessible à `https://<server>:8443`. Connectez-vous avec les identifiants administrateur initiaux :

    - **Nom d'utilisateur** : `admin`
    - **Mot de passe** : Imprimé sur la carte de démarrage rapide du produit, ou affiché à la fin de l'installateur

    <Warning>
      Changez immédiatement le mot de passe par défaut. Les identifiants par défaut sont les mêmes pour tous les déploiements VaultWares et sont publiquement documentés.
    </Warning>
  </Step>
  <Step title="Changer le mot de passe administrateur par défaut">
    ```bash
    vw-admin password change \
      --user admin \
      --current-password <default-password> \
      --new-password <strong-new-password>
    ```
  </Step>
  <Step title="Créer des comptes administrateurs nommés">
    Évitez d'utiliser le compte générique `admin` pour les opérations quotidiennes. Créez des comptes nommés pour chaque administrateur :

    ```bash
    # Create an administrator account
    vw-admin user create \
      --username alice \
      --email alice@example.com \
      --role Administrator \
      --require-mfa true \
      --send-welcome-email

    # Create a security officer account (for HSM operations)
    vw-admin user create \
      --username bob-so \
      --email bob@example.com \
      --role SecurityOfficer \
      --require-mfa true \
      --require-hardware-key true
    ```
  </Step>
  <Step title="Désactiver ou restreindre le compte administrateur par défaut">
    ```bash
    # Lock the default admin account after named accounts are configured
    vw-admin user disable --username admin

    # Or restrict to console access only
    vw-admin user restrict \
      --username admin \
      --allow-console-only \
      --require-mfa true
    ```
  </Step>
</Steps>

## Étape 2 : Activation de la licence

```bash
# Activate license online
vw-admin license activate --key VWXX-XXXX-XXXX-XXXX-XXXX

# Verify license status
vw-admin license status

# Output:
# Product:         VaultAccess Enterprise
# License key:     VWAC-XXXX-XXXX-XXXX-XXXX
# Status:          Active
# Seats licensed:  500
# Seats in use:    0
# Valid until:     2026-12-31
# Features:        RBAC, ABAC, PAM, SSO, SIEM integration

# Offline activation (for air-gapped environments)
vw-admin license request-offline-token --key VWXX-XXXX-XXXX-XXXX-XXXX
# Submit the token at https://portal.vaultwares.com/offline-activation
# Then apply the activation response:
vw-admin license apply-offline-response --file activation-response.json
```

## Étape 3 : Configuration du certificat TLS

Remplacez le certificat auto-signé installé par défaut par un certificat signé par une Autorité de Certification (AC).

```bash
# Generate a CSR
vw-admin tls generate-csr \
  --common-name vaultaccess.corp.example.com \
  --san "DNS:vaultaccess.corp.example.com,DNS:vault.corp.example.com" \
  --key-size 2048 \
  --output /etc/vaultwares/tls/server.csr

# After obtaining the signed certificate from your CA:
vw-admin tls install \
  --cert /etc/vaultwares/tls/server.crt \
  --key /etc/vaultwares/tls/server.key \
  --ca-chain /etc/vaultwares/tls/ca-chain.crt

# Restart services to apply
vw-admin service restart --all
```

## Étape 4 : Intégration Active Directory / LDAP

<Tabs>
  <Tab title="Active Directory">
    ```bash
    # Test AD connectivity before joining
    vw-admin ad test-connection \
      --domain corp.example.com \
      --dc dc01.corp.example.com

    # Join the domain
    vw-admin ad join \
      --domain corp.example.com \
      --dc dc01.corp.example.com \
      --username CORP\\svc_vaultaccess \
      --password <service-account-password> \
      --ou "OU=VaultWares,DC=corp,DC=example,DC=com"

    # Configure group synchronization
    vw-admin ad configure-sync \
      --base-dn "DC=corp,DC=example,DC=com" \
      --user-filter "(&(objectClass=user)(memberOf=CN=VaultWaresUsers,OU=Groups,DC=corp,DC=example,DC=com))" \
      --group-sync true \
      --sync-interval 60m \
      --nested-groups true

    # Run initial sync
    vw-admin ad sync --force
    ```
  </Tab>
  <Tab title="LDAP">
    ```bash
    # Configure LDAP connection
    vw-admin ldap configure \
      --url ldap://ldap.example.com:389 \
      --bind-dn "cn=svc_vaultaccess,ou=services,dc=example,dc=com" \
      --bind-password <bind-password> \
      --base-dn "dc=example,dc=com" \
      --user-search-filter "(objectClass=inetOrgPerson)" \
      --group-search-filter "(objectClass=groupOfNames)" \
      --tls-ca-cert /etc/vaultwares/tls/ldap-ca.crt \
      --start-tls true

    # Test LDAP configuration
    vw-admin ldap test --user alice

    # Enable LDAP authentication
    vw-admin ldap enable
    ```
  </Tab>
</Tabs>

## Étape 5 : Configuration des notifications par courriel

```bash
# Configure SMTP for email notifications
vw-admin email configure \
  --smtp-host smtp.example.com \
  --smtp-port 587 \
  --smtp-tls starttls \
  --smtp-username noreply@example.com \
  --smtp-password <smtp-password> \
  --from-address "VaultWares Alerts <noreply@example.com>" \
  --from-name "VaultWares"

# Send a test email
vw-admin email test --recipient admin@example.com

# Configure notification recipients
vw-admin notifications configure \
  --security-alerts security@example.com \
  --system-alerts sysadmin@example.com \
  --compliance-reports compliance@example.com \
  --backup-reports backup@example.com
```

## Étape 6 : Configuration de la sauvegarde initiale

Configurez au moins une destination de sauvegarde avant que le système ne soit mis en production.

```bash
# Add a local backup repository
vw-backup repo add \
  --name local-nas \
  --type nfs \
  --path //nas01.corp.example.com/vaultbackup \
  --encrypt-key hsm:Backup-Master-Key

# Add a cloud backup repository
vw-backup repo add \
  --name aws-s3-backup \
  --type s3 \
  --bucket corp-vaultbackup-vault \
  --region us-east-1 \
  --access-key $AWS_ACCESS_KEY_ID \
  --secret-key $AWS_SECRET_ACCESS_KEY \
  --encrypt-key hsm:Backup-Master-Key \
  --immutable-retention-days 30

# Create and apply initial backup schedule
vw-backup schedule create \
  --name SystemConfiguration \
  --source /etc/vaultwares \
  --destination local-nas \
  --type full \
  --cron "0 3 * * *" \
  --retention-days 90
```

## Étape 7 : Intégration HSM

Si vous disposez d'un VaultHSM, intégrez-le avant de créer des clés de production.

```bash
# Add HSM to the key management configuration
vw-admin hsm add \
  --name production-hsm \
  --host hsm01.corp.example.com \
  --slot 0 \
  --pin <hsm-user-pin> \
  --tls-cert /etc/vaultwares/tls/hsm-client.crt \
  --tls-key /etc/vaultwares/tls/hsm-client.key

# Verify HSM connectivity and generate master key
vw-admin hsm test --name production-hsm

# Generate system master key on HSM
vw-admin hsm generate-master-key \
  --hsm production-hsm \
  --label "VaultWares-Master-Key-v1" \
  --algorithm AES-256
```

```bash
# Run the built-in configuration health check
vw-admin health-check --full

# Output:

# [✓] TLS certificate: Valid (CN=vaultaccess.corp.example.com)
# [✓] Database: Connected, 0ms latency
# [✓] Active Directory: Connected (dc01.corp.example.com), 15ms latency
# [✓] SMTP: Reachable (smtp.example.com:587)
# [✓] HSM: Connected (hsm01.corp.example.com, FIPS 140-2 Level 3)
# [✓] Backup repository: Accessible (nas01, 45ms)
# [✓] Admin accounts: 2 named administrators, default admin disabled
# [!] MFA: 1 administrator account without MFA enrolled (bob@example.com)

# Address any warnings before going live

```
