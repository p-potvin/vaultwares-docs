## Vue d'ensemble

Les Modules de sécurité matériels (HSM) VaultWares fournissent un matériel dédié et résistant aux manipulations pour les opérations cryptographiques et la gestion des clés. Les HSM protègent vos clés cryptographiques les plus sensibles et effectuent les opérations de chiffrement, déchiffrement, de signature et d'authentification à haute vitesse.

## Fonctionnalités clés

### Opérations cryptographiques
- **Génération de clés** : RSA, ECC, AES, DES/3DES
- **Signatures numériques** : RSA, ECDSA, EdDSA
- **Chiffrement/déchiffrement** : Algorithmes symétriques et asymétriques
- **Hachage** : Famille SHA-2, SHA-3
- **Dérivation de clés** : PBKDF2, HKDF, X9.63
- **Génération de nombres aléatoires** : RNG matériel véritable (TRNG)

### Fonctionnalités de sécurité
- **Détection de sabotage** : L'intrusion physique déclenche la désactivation automatique des clés (zeroization)
- **Stockage sécurisé des clés** : Les clés ne quittent jamais l'HSM en texte clair
- **Accès basé sur les rôles** : Séparation des tâches avec plusieurs rôles d'administrateur
- **Journalisation d'audit** : Journaux d'opérations cryptographiques complets
- **Sauvegarde et récupération** : Sauvegarde sécurisée des clés avec division de clés M-sur-N
- **Conformité FIPS** : Modules cryptographiques certifiés

### Haute disponibilité
- **Clustering** : Configurations actif-actif ou actif-passif
- **Équilibrage de charge** : Distribution des opérations sur plusieurs HSM
- **Basculement automatique** : Fonctionnement continu en cas de défaillance matérielle
- **Remplacement à chaud** : Remplacer les unités défaillantes sans temps d'arrêt (modèles réseau)

## Cas d'utilisation

### PKI et gestion des certificats
```bash
# Generate CA root key
vw-hsm generate-key \
  --type rsa \
  --size 4096 \
  --label "Root-CA-Key" \
  --extractable false

# Sign certificate request
vw-hsm sign \
  --key "Root-CA-Key" \
  --input csr.pem \
  --output cert.pem \
  --algorithm sha256-rsa
```

### Signature de code
```bash
# Sign application binary
vw-hsm code-sign \
  --key "Code-Signing-Key" \
  --input application.exe \
  --output application-signed.exe \
  --timestamp https://timestamp.vaultwares.com
```

### Chiffrement de bases de données
```bash
# Generate database master key
vw-hsm generate-key \
  --type aes \
  --size 256 \
  --label "DB-Master-Key" \
  --extractable false

# Encrypt data encryption key
vw-hsm encrypt \
  --key "DB-Master-Key" \
  --input dek.bin \
  --output dek-encrypted.bin
```

### Déchargement SSL/TLS
```nginx
# Nginx configuration with HSM
ssl_engine vaulthsm;
ssl_certificate /etc/ssl/certs/server.crt;
ssl_certificate_key "pkcs11:token=VaultHSM;object=WebServer-Key";
ssl_protocols TLSv1.2 TLSv1.3;
```

## Installation et configuration

### Installation initiale

<Steps>
  <Step title="Installer le matériel">
    Pour les modèles PCIe, installez la carte dans un emplacement PCIe disponible. Pour les modèles réseau, montez-le dans un rack et connectez les câbles d'alimentation et de réseau.
  </Step>
  <Step title="Initialiser l'HSM">
    ```bash
    vw-hsm init \
      --admin-pin <secure-pin> \
      --label "Production-HSM-01"
    ```
  </Step>
  <Step title="Créer l'agent de sécurité">
    ```bash
    vw-hsm create-user \
      --role security-officer \
      --username so-admin \
      --pin <so-pin>
    ```
  </Step>
  <Step title="Configurer le réseau (modèles réseau)">
    ```bash
    vw-hsm network-config \
      --ip 192.168.1.100 \
      --netmask 255.255.255.0 \
      --gateway 192.168.1.1
    ```
  </Step>
</Steps>

### Configuration de haute disponibilité

```bash
# Configure HSM cluster
vw-hsm cluster create \
  --name production-cluster \
  --mode active-active \
  --members hsm01.example.com,hsm02.example.com,hsm03.example.com

# Enable automatic failover
vw-hsm cluster failover \
  --enable \
  --health-check-interval 10s \
  --failover-threshold 3

# Synchronize keys across cluster
vw-hsm cluster sync \
  --all-keys
```

## Gestion des clés

### Génération de clés

<CodeGroup>
```bash RSA key pair
vw-hsm generate-key \
  --type rsa \
  --size 2048 \
  --label "MyRSAKey" \
  --usage sign,decrypt \
  --extractable false
```

```bash ECC key pair
vw-hsm generate-key \
  --type ecc \
  --curve secp256r1 \
  --label "MyECCKey" \
  --usage sign \
  --extractable false
```

```bash AES symmetric key
vw-hsm generate-key \
  --type aes \
  --size 256 \
  --label "MyAESKey" \
  --usage encrypt,decrypt \
  --extractable false
```
</CodeGroup>

### Sauvegarde et récupération des clés

```bash
# Create key backup with M-of-N key splitting
vw-hsm backup create \
  --output backup.enc \
  --split 3-of-5 \
  --custodians alice,bob,charlie,david,eve

# Restore from backup (requires 3 custodian keys)
vw-hsm backup restore \
  --input backup.enc \
  --custodian-keys alice.key,bob.key,charlie.key
```

### Rotation des clés

```bash
# Rotate encryption key
vw-hsm key-rotate \
  --old-key "DB-Master-Key-v1" \
  --new-key "DB-Master-Key-v2" \
  --re-encrypt-data /data/encrypted/*
```

## Exemples d'intégration

### Intégration PKCS#11

```c
// C/C++ application using PKCS#11
#include <pkcs11.h>

CK_FUNCTION_LIST *functions;
CK_SESSION_HANDLE session;

// Load VaultHSM PKCS#11 library
C_GetFunctionList(&functions);
functions->C_Initialize(NULL);

// Open session
CK_SLOT_ID slot = 0;
functions->C_OpenSession(slot, CKF_SERIAL_SESSION | CKF_RW_SESSION,
                         NULL, NULL, &session);

// Login
CK_UTF8CHAR pin[] = "your-pin";
functions->C_Login(session, CKU_USER, pin, sizeof(pin) - 1);

// Perform cryptographic operations
// ...

functions->C_Logout(session);
functions->C_CloseSession(session);
```

### Intégration Java

```java
import java.security.*;
import javax.crypto.*;

// Configure Java to use VaultHSM
String configName = "vaulthsm.cfg";
Provider hsm = new sun.security.pkcs11.SunPKCS11(configName);
Security.addProvider(hsm);

// Get key from HSM
KeyStore ks = KeyStore.getInstance("PKCS11", hsm);
ks.load(null, pin);
PrivateKey key = (PrivateKey) ks.getKey("MyRSAKey", null);

// Sign data
Signature sig = Signature.getInstance("SHA256withRSA", hsm);
sig.initSign(key);
sig.update(data);
byte[] signature = sig.sign();
```

### Intégration OpenSSL

```bash
# OpenSSL configuration for VaultHSM
openssl_conf = openssl_init

[openssl_init]
engines = engine_section

[engine_section]
pkcs11 = pkcs11_section

[pkcs11_section]
engine_id = pkcs11
MODULE_PATH = /usr/lib/vaulthsm-pkcs11.so

# Use HSM key with OpenSSL
openssl dgst -sha256 -sign "pkcs11:token=VaultHSM;object=MyKey" \
  -out signature.bin data.txt
```

## Surveillance et audit

### Activer la journalisation d'audit

```bash
# Configure comprehensive audit logging
vw-hsm audit-config \
  --enable \
  --log-level detailed \
  --syslog-server syslog.example.com:514 \
  --log-operations all \
  --include-failed-attempts
```

### Afficher les journaux d'audit

```bash
# Query audit logs
vw-hsm audit-query \
  --start-date "2024-01-01" \
  --end-date "2024-01-31" \
  --operation sign \
  --user alice \
  --format json > audit-report.json
```

### Surveillance des performances

```bash
# Real-time performance metrics
vw-hsm monitor \
  --metrics operations,latency,queue-depth \
  --interval 5s

# Output:
# Operations/sec: 15,234
# Avg latency: 2.3ms
# Queue depth: 12
```

## Dépannage

<AccordionGroup>
  <Accordion title="HSM non détecté">
    - Vérifier les connexions d'alimentation (modèles réseau)
    - Vérifier la compatibilité du slot PCIe (modèles PCIe)
    - S'assurer que les pilotes sont installés : `vw-hsm-driver --version`
    - Vérifier les journaux système : `journalctl -u vaulthsm`
  </Accordion>

  <Accordion title="Dégradation des performances">
    - Vérifier l'utilisation du CPU et de la mémoire sur l'hôte
    - Vérifier la latence réseau (modèles réseau)
    - Examiner la profondeur de la file d'attente : `vw-hsm status --verbose`
    - Envisager d'ajouter des HSM au cluster pour la distribution de charge
  </Accordion>

  <Accordion title="Échecs d'authentification">
    - Vérifier que le PIN/mot de passe est correct
    - Vérifier si le compte est verrouillé : `vw-hsm user-status`
    - Examiner les journaux d'audit pour les tentatives échouées
    - Réinitialiser le PIN de l'utilisateur si nécessaire (nécessite des privilèges d'agent de sécurité)
  </Accordion>

  <Accordion title="Problèmes de synchronisation du cluster">
    - Vérifier la connectivité réseau entre les membres du cluster
    - Vérifier la synchronisation temporelle (NTP)
    - Examiner l'état du cluster : `vw-hsm cluster status`
    - Forcer la resynchronisation : `vw-hsm cluster sync --force`
  </Accordion>
</AccordionGroup>
