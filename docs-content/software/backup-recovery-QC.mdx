## Aperçu

VaultBackup offre une sauvegarde et une reprise après sinistre automatisées et chiffrées par matériel pour les organisations qui ne peuvent pas se permettre de perdre des données. Toutes les données de sauvegarde sont chiffrées à la source à l'aide d'AES-256 avant la transmission, et VaultBackup s'intègre nativement à VaultHSM pour la gestion des clés d'entreprise. La protection intégrée contre les rançongiciels et le stockage de sauvegarde immuable garantissent que vos données de récupération restent sécurisées même lorsque votre environnement principal est compromis.

## Fonctionnalités clés

### Chiffrement à chaque étape

- **Chiffrement à la source**: Les données sont chiffrées sur l'agent de sauvegarde avant de quitter le système source
- **Chiffrement en transit**: Tous les flux de sauvegarde utilisent TLS 1.3
- **Chiffrement au repos**: Les référentiels de sauvegarde utilisent AES-256-XTS
- **Gestion des clés**: Intégration avec VaultHSM pour les clés de sauvegarde protégées par matériel
- **Séquestre de clés**: Copies de clés chiffrées détenues en séquestre sécurisé pour les scénarios de sinistre

### Types de sauvegarde

<Tabs>
  <Tab title="Sauvegarde complète">
    Une copie complète de toutes les données sélectionnées. Utilisée comme base pour les chaînes incrémentielles et différentielles.

    ```bash
    # Run a full backup immediately
    vw-backup run \
      --type full \
      --source /data/production \
      --destination backup://nas01/vaults/prod-full \
      --encrypt-key hsm:DB-Backup-Key \
      --compress lz4
    ```
  </Tab>
  <Tab title="Sauvegarde incrémentielle">
    Sauvegarde uniquement des données modifiées depuis la dernière sauvegarde de tout type. Minimise la fenêtre de sauvegarde et la consommation de stockage.

    ```bash
    # Run an incremental backup
    vw-backup run \
      --type incremental \
      --source /data/production \
      --destination backup://nas01/vaults/prod-incr \
      --base-backup latest \
      --encrypt-key hsm:DB-Backup-Key
    ```
  </Tab>
  <Tab title="Sauvegarde différentielle">
    Sauvegarde toutes les données modifiées depuis la dernière sauvegarde **complète**. Restauration plus rapide que l'incrémentielle au prix d'un plus grand stockage.

    ```bash
    # Run a differential backup
    vw-backup run \
      --type differential \
      --source /data/production \
      --destination backup://nas01/vaults/prod-diff \
      --full-backup-ref "2024-11-01T02:00:00Z" \
      --encrypt-key hsm:DB-Backup-Key
    ```
  </Tab>
</Tabs>

```json
{
  "schedule": {
    "name": "ProductionBackupSchedule",
    "timezone": "America/New_York",
    "jobs": [
      {
        "name": "Daily incremental",

        "encrypt_key": "hsm:DB-Backup-Key",
        "compression": "lz4",
        "verify_after": true
      },
      {
        "name": "Weekly full",
        "type": "full",
        "cron": "0 1 * * 0",
        "retention_days": 90,
        "source": "/data/production",
        "destination": "backup://nas01/vaults/prod",
        "encrypt_key": "hsm:DB-Backup-Key",
        "compression": "zstd",
        "replicate_to": "backup://cloud-vault-us-east-1/vaults/prod",
        "verify_after": true
      },
      {
        "name": "Monthly archive",
        "type": "full",
        "cron": "0 0 1 * *",
        "retention_days": 2555,
        "source": "/data/production",
        "destination": "backup://cloud-vault-us-east-1/vaults/prod-archive",
        "encrypt_key": "hsm:DB-Backup-Key",
        "immutable": true,
        "verify_after": true
      }
    ]
  }
}
```

```bash
# Apply schedule configuration
vw-backup schedule apply --config /etc/vaultbackup/schedules/production.json

# List active schedules
vw-backup schedule list

# Run a scheduled job manually
vw-backup schedule run-now --job "Daily incremental"
```

### Réplication hors site

```bash
# Configure off-site replication to AWS S3
vw-backup replication configure \
  --name "AWS-US-East-1" \
  --type s3 \
  --bucket my-vaultbackup-vault \
  --region us-east-1 \
  --access-key $AWS_ACCESS_KEY_ID \
  --secret-key $AWS_SECRET_ACCESS_KEY \
  --encrypt-in-transit true \
  --bandwidth-limit 100Mbps

# Configure replication to Azure Blob Storage
vw-backup replication configure \
  --name "Azure-East-US" \
  --type azure-blob \
  --account my-backup-storage \
  --container vaultbackup \
  --sas-token $AZURE_SAS_TOKEN

# View replication status
vw-backup replication status --all
```

### Protection contre les rançongiciels

VaultBackup Entreprise comprend une protection multi-couches contre les rançongiciels :

<AccordionGroup>
  <Accordion title="Stockage de sauvegarde immuable">
    Les données de sauvegarde écrites sur un stockage immuable ne peuvent être modifiées ou supprimées pendant la période de rétention configurée, même par des administrateurs disposant de pleins privilèges. Pris en charge sur VaultWares Cloud Vault et le stockage compatible S3 avec Object Lock.

    ```bash
    # Enable immutability on a backup repository
    vw-backup repo set-immutable \
      --repo backup://cloud-vault-us-east-1/vaults/prod \
      --retention-days 30 \
      --compliance-mode governance
    ```
  </Accordion>
  <Accordion title="Surveillance des fichiers leurres (Honeypot)">
    VaultBackup place des fichiers sentinelles cachés dans les répertoires surveillés. Toute modification de ces fichiers déclenche une alerte immédiate et met optionnellement en pause les nouvelles sauvegardes pour empêcher le chiffrement des données déjà rançonnées.

    ```bash
    # Enable honeypot monitoring
    vw-backup ransomware-protection enable \
      --directories /data/production,/data/databases \
      --alert-on-trigger \
      --pause-backup-on-trigger true
    ```
  </Accordion>
  <Accordion title="Analyse d'entropie">
    Les flux de sauvegarde sont analysés pour une entropie anormalement élevée, ce qui est caractéristique des données chiffrées par rançongiciel. Les sauvegardes suspectes sont signalées pour examen avant d'écraser les sauvegardes propres précédentes.
  </Accordion>
</AccordionGroup>

## Procédures de récupération

### Restaurer à partir d'une sauvegarde

<Steps>
  <Step title="Lister les points de récupération disponibles">
    ```bash
    # List all recovery points for a source
    vw-backup restore list \
      --source /data/production \
      --repo backup://nas01/vaults/prod

    # Output:
    # Recovery Point           Type          Size     Encrypted  Verified
    # 2024-11-15T02:00:01Z     Incremental   45.2 GB  Yes        Yes
    # 2024-11-14T02:00:02Z     Incremental   44.8 GB  Yes        Yes
    # 2024-11-10T01:00:05Z     Full          312 GB   Yes        Yes
    ```
  </Step>
  <Step title="Vérifier l'intégrité de la sauvegarde">
    ```bash
    # Verify a specific recovery point
    vw-backup verify \
      --repo backup://nas01/vaults/prod \
      --recovery-point "2024-11-15T02:00:01Z" \
      --decrypt-key hsm:DB-Backup-Key
    ```
  </Step>
  <Step title="Effectuer la restauration">
    ```bash
    # Restore to original location
    vw-backup restore \
      --repo backup://nas01/vaults/prod \
      --recovery-point "2024-11-15T02:00:01Z" \
      --destination /data/production \
      --decrypt-key hsm:DB-Backup-Key \
      --overwrite-mode prompt

    # Restore to alternate location (for testing)
    vw-backup restore \
      --repo backup://nas01/vaults/prod \
      --recovery-point "2024-11-15T02:00:01Z" \
      --destination /data/restore-test \
      --decrypt-key hsm:DB-Backup-Key
    ```
  </Step>
  <Step title="Vérifier les données restaurées">
    ```bash
    # Generate a checksum report of restored files
    vw-backup verify-restore \
      --source /data/restore-test \
      --compare-to-backup \
      --recovery-point "2024-11-15T02:00:01Z"
    ```
  </Step>
</Steps>

### Reprise après sinistre

```bash
# Initiate a full DR failover
vw-backup dr failover \
  --plan DR-Production \
  --target-site aws-us-west-2 \
  --rpo-target 1h \
  --rto-target 4h \
  --dry-run false

# Monitor DR failover progress
vw-backup dr status --plan DR-Production

# Test DR plan (non-destructive)
vw-backup dr test \
  --plan DR-Production \
  --isolated-network \
  --snapshot-only
```

## Vérification de la sauvegarde

Une vérification régulière est essentielle. VaultBackup prend en charge la vérification à la demande et planifiée :

```bash
# Verify all backups in a repository
vw-backup verify-all \
  --repo backup://nas01/vaults/prod \
  --decrypt-key hsm:DB-Backup-Key \
  --threads 4

# Schedule automated weekly verification
vw-backup schedule add-verification \
  --repo backup://nas01/vaults/prod \
  --cron "0 3 * * 0" \
  --alert-on-failure \
  --alert-email security@example.com
```
