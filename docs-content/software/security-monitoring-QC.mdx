## Vue d'ensemble

VaultMonitor est la plateforme unifiée de surveillance de la sécurité de VaultWares. Elle collecte la télémétrie de tous les produits matériels et logiciels VaultWares, corrèle les événements dans votre environnement, et affiche des alertes exploitables en temps réel. VaultMonitor est livré avec des intégrations SIEM prêtes à l'emploi pour Splunk, IBM QRadar, Microsoft Sentinel, et d'autres, ainsi qu'un tableau de bord intégré pour les organisations qui n'ont pas de SIEM existant.

## Fonctionnalités clés

### Collecte d'événements en temps réel

VaultMonitor collecte les événements de tous les produits VaultWares et de sources externes :

```bash
# View current ingestion status
vw-monitor status --ingestion

# Windows Event Log (DC-01)    1,103        Active
# Syslog (firewall)            2,871        Active
# ---
# Total                        11,060       Healthy
```

### Règles de détection des menaces

VaultMonitor est livré avec plus de 500 règles de détection pré-construites, adaptées aux environnements VaultWares. Les règles personnalisées utilisent un langage de règles simple basé sur YAML :

```yaml
# Custom detection rule: Brute-force attack on VaultDrive fleet
name: VaultDrive_BruteForce_Attack
description: >
  Detects repeated failed authentication attempts on VaultDrive devices,
  indicating a potential brute-force attack.
severity: high
tags: [brute-force, encrypted-storage, lateral-movement]

detection:
  source: vaultdrive.authentication
  filter:
    event_type: auth_failure
  groupby:
    - device_id
    - source_ip
  threshold:
    count: 5
    timeframe: 5m

response:
  alert: true
  severity: high
  actions:
    - type: notify
      channels: [email, slack, pagerduty]
    - type: api_call
      endpoint: https://api.vaultwares.com/v1/devices/{device_id}/lock
      method: POST
    - type: create_ticket
      integration: jira
      project: SEC
      priority: P1
```

```bash
# Apply a custom rule
vw-monitor rules apply --file /etc/vaultmonitor/rules/brute-force.yaml

# List all active rules
vw-monitor rules list --status active

# Test a rule against historical data
vw-monitor rules test \
  --rule VaultDrive_BruteForce_Attack \
  --timeframe "2024-11-01T00:00:00Z/2024-11-15T23:59:59Z" \
  --dry-run
```

### Détection d'anomalies

L'édition Entreprise comprend la détection d'anomalies basée sur l'apprentissage automatique :

<AccordionGroup>
  <Accordion title="Analyse du comportement utilisateur (ACU)">
    Établit une ligne de base comportementale pour chaque utilisateur et alerte lorsque l'activité s'en écarte de manière significative. Détecte le vol d'identifiants, les menaces internes et le compromis de compte.

    ```bash
    # View anomaly scores for users
    vw-monitor anomaly user-scores \
      --top 10 \
      --timeframe 24h \
      --min-score 70

    # Output:
    # User                 Score  Anomalies
    # charlie@example.com  94     Off-hours HSM access, new device
    # dave@example.com     78     Bulk file exports from VaultDrive
    ```
  </Accordion>
  <Accordion title="Analyse du comportement des appareils">
    Surveille la télémétrie des appareils à la recherche de modèles inhabituels : volumes de données anormaux, emplacements géographiques inattendus, séquences d'opérations inhabituelles.
  </Accordion>
  <Accordion title="Analyse du trafic réseau">
    S'intègre à VaultGate pour détecter les flux réseau anormaux, le signalement de type commande et contrôle (C2), et les schémas d'exfiltration de données.
  </Accordion>
</AccordionGroup>

### Configuration des alertes

```json
{
  "alert_policy": {
    "name": "CriticalAlertPolicy",
    "channels": {
      "email": {
        "recipients": ["security@example.com", "soc@example.com"],
        "min_severity": "medium",
        "digest_interval_minutes": 60
      },
      "pagerduty": {
        "service_key": "pd_svc_key_here",
        "min_severity": "high",
        "escalation_policy": "24x7-Security"
      },
      "slack": {
        "webhook_url": "https://hooks.slack.com/services/...",
        "channel": "#security-alerts",
        "min_severity": "medium"
      },
      "webhook": {
        "url": "https://soar.example.com/api/incidents",
        "method": "POST",
        "headers": {"Authorization": "Bearer soar-token"},
        "min_severity": "high"
      }
    }
  }
}
```

```bash
# Apply alert policy
vw-monitor alerts set-policy --config /etc/vaultmonitor/alerts.json

# Test alert delivery
vw-monitor alerts test --channel email --recipient security@example.com

# Acknowledge an alert
vw-monitor alerts acknowledge --id alert_abc123 --comment "Investigating"

# Escalate an alert
vw-monitor alerts escalate --id alert_abc123 --to soc-lead@example.com
```

## Intégration SIEM

### Splunk

<Tabs>
  <Tab title="Splunk HEC">
    ```bash
    # Configure Splunk HEC output
    vw-monitor siem configure splunk \
      --type hec \
      --url https://splunk.example.com:8088 \
      --token <splunk-hec-token> \
      --index vaultwares \
      --sourcetype vaultwares:events \
      --tls-verify true

    # Verify connectivity
    vw-monitor siem test --integration splunk
    ```
  </Tab>
  <Tab title="Forwarder Splunk">
    ```bash
    # Forward events via Syslog to Splunk Universal Forwarder
    vw-monitor siem configure syslog \
      --host splunk-uf.example.com \
      --port 5514 \
      --protocol tcp \
      --format rfc5424 \
      --tls true
    ```
  </Tab>
</Tabs>

### IBM QRadar

```bash
# Configure QRadar DSM integration
vw-monitor siem configure qradar \
  --host qradar.example.com \
  --port 514 \
  --protocol syslog-tls \
  --log-source-type "VaultWares Monitor" \
  --device-support-module /opt/vaultmonitor/dsm/vaultwares-qradar-dsm.zip
```

### Microsoft Sentinel

```bash
# Configure Azure Sentinel via Log Analytics workspace
vw-monitor siem configure sentinel \
  --workspace-id <log-analytics-workspace-id> \
  --shared-key <primary-key> \
  --table-name VaultWaresEvents \
  --batch-size 500
```

### Syslog / CEF générique

```bash
# Configure CEF output for any SIEM
vw-monitor siem configure cef \
  --host siem.example.com \
  --port 514 \
  --protocol udp \
  --vendor VaultWares \
  --product VaultMonitor \
  --version 3.0
```

## Gestion des journaux

```bash
# Query events with filters
vw-monitor query \
  --source vaultdrive \
  --event-type auth_failure \
  --start "2024-11-15T00:00:00Z" \
  --end "2024-11-15T23:59:59Z" \
  --user alice@example.com \
  --format json \
  --limit 100

# Export logs for forensic analysis
vw-monitor export \
  --start "2024-11-01T00:00:00Z" \
  --end "2024-11-15T23:59:59Z" \
  --sources all \
  --format json \
  --output /forensics/nov2024-export.json.gz \
  --sign-with hsm:AuditLog-Signing-Key
```

<Note>
  Les journaux exportés sont signés numériquement par défaut lorsqu'une clé de signature VaultHSM est configurée, fournissant une preuve cryptographique de l'intégrité des journaux à des fins forensiques et de conformité.
</Note>
