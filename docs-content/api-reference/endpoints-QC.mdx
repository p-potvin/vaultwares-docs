## Aperçu

L'API REST de VaultWares fournit un accès programmatique à toutes les fonctions de gestion. Utilisez-la pour automatiser la gestion des appareils, appliquer des politiques, gérer les utilisateurs, interroger les journaux d'audit et gérer les clés cryptographiques et les certificats.

**URL de base**: `https://api.vaultwares.com/v1`

Toutes les requêtes doivent être authentifiées. Consultez le [guide d'authentification](/api-reference/authentication) pour plus de détails.

## Conventions courantes

- Tous les corps de requête et de réponse sont au format JSON (`Content-Type: application/json`)
- Les horodatages sont au format ISO 8601 (`2024-11-15T14:30:00Z`)
- La pagination utilise les paramètres `limit` et `cursor`
- Les erreurs suivent un format cohérent de `{"error": {"code": "...", "message": "...", "type": "..."}}`

## Appareils

### Lister les appareils

```http
GET /v1/devices
```

| Paramètre | Type | Description |
|-----------|------|-------------|
| `type` | string | Filtrer par type d'appareil : `vaultdrive`, `vaulthsm`, `vaultscan`, `vaultgate` |
| `status` | string | Filtrer par statut : `online`, `offline`, `locked`, `error` |
| `user_id` | string | Filtrer par utilisateur assigné |
| `limit` | integer | Résultats par page (par défaut : 20, max : 100) |
| `cursor` | string | Curseur de pagination de la réponse précédente |

```bash
curl https://api.vaultwares.com/v1/devices \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -G \
  --data-urlencode "type=vaultdrive" \
  --data-urlencode "status=online" \
  --data-urlencode "limit=50"
```

```json
{
  "data": [
    {
      "id": "dev_VD-ENT-00247",
      "type": "vaultdrive",
      "model": "VaultDrive Enterprise",
      "serial": "VD-ENT-00247",
      "firmware_version": "3.2.0",
      "status": "online",
      "assigned_to": "user_alice123",
      "last_seen": "2024-11-15T14:29:55Z",
      "policy": "corporate-standard",
      "encrypted": true,
      "capacity_gb": 1024
    }
  ],
  "pagination": {
    "cursor": "cursor_abc123",
    "has_more": true,
    "total": 247
  }
}
```

### Récupérer un appareil

```http
GET /v1/devices/{device_id}
```

### Mettre à jour un appareil

```http
PATCH /v1/devices/{device_id}
```

```bash
curl -X PATCH https://api.vaultwares.com/v1/devices/dev_VD-ENT-00247 \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "policy": "high-security",
    "assigned_to": "user_bob456",
    "metadata": {"department": "Finance", "asset_tag": "IT-00247"}
  }'
```

### Verrouiller / déverrouiller un appareil

```bash
# Lock device
curl -X POST https://api.vaultwares.com/v1/devices/dev_VD-ENT-00247/lock \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -d '{"reason": "Suspected loss"}'

# Unlock device
curl -X POST https://api.vaultwares.com/v1/devices/dev_VD-ENT-00247/unlock \
  -H "X-API-Key: vw_live_1234567890abcdef"
```

### Effacer cryptographiquement un appareil

```bash
curl -X POST https://api.vaultwares.com/v1/devices/dev_VD-ENT-00247/crypto-erase \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -d '{
    "confirm": true,
    "reason": "GDPR right-to-erasure request",
    "audit_reference": "DSR-2024-1115-001"
  }'
```

<Warning>
  Le point de terminaison `crypto-erase` est irréversible. Toutes les données sur l'appareil sont détruites de façon permanente en effaçant la clé de chiffrement.
</Warning>

## Politiques

### Lister les politiques

```http
GET /v1/policies
```

### Créer une politique

```http
POST /v1/policies
```

```bash
curl -X POST https://api.vaultwares.com/v1/policies \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "high-security",
    "description": "Policy for devices handling sensitive data",
    "settings": {
      "min_password_length": 16,
      "require_2fa": true,
      "auto_lock_minutes": 5,
      "max_failed_attempts": 5,
      "audit_logging": "all",
      "require_mfa_for_unlock": true
    }
  }'
```

### Déployer une politique sur les appareils

```bash
curl -X POST https://api.vaultwares.com/v1/policies/pol_abc123/deploy \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -d '{"device_ids": ["dev_VD-ENT-00247", "dev_VD-ENT-00248"], "force": false}'
```

## Utilisateurs

### Lister les utilisateurs

```http
GET /v1/users
```

### Créer un utilisateur

```http
POST /v1/users
```

```bash
curl -X POST https://api.vaultwares.com/v1/users \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "charlie@example.com",
    "first_name": "Charlie",
    "last_name": "Brown",
    "roles": ["StandardUser"],
    "department": "Engineering",
    "mfa_required": true,
    "send_welcome_email": true
  }'
```

```json
{
  "id": "user_charlie789",
  "email": "charlie@example.com",
  "first_name": "Charlie",
  "last_name": "Brown",
  "status": "active",
  "roles": ["StandardUser"],
  "mfa_enrolled": false,
  "created_at": "2024-11-15T14:30:00Z"
}
```

### Révoquer toutes les sessions utilisateur

```bash
curl -X POST https://api.vaultwares.com/v1/users/user_charlie789/revoke-sessions \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -d '{"reason": "Security incident"}'
```

## Journaux d'audit

### Interroger les journaux d'audit

```http
GET /v1/audit
```

| Paramètre | Type | Description |
|-----------|------|-------------|
| `start` | ISO 8601 | Début de la plage de temps |
| `end` | ISO 8601 | Fin de la plage de temps |
| `user_id` | string | Filtrer par utilisateur |
| `device_id` | string | Filtrer par appareil |
| `event_type` | string | Filtrer par type d'événement |
| `limit` | integer | Résultats par page (max : 1000) |

```bash
curl https://api.vaultwares.com/v1/audit \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -G \
  --data-urlencode "start=2024-11-15T00:00:00Z" \
  --data-urlencode "end=2024-11-15T23:59:59Z" \
  --data-urlencode "event_type=auth_failure" \
  --data-urlencode "limit=100"
```

## Clés

### Lister les clés

```http
GET /v1/keys
```

### Générer une clé

```http
POST /v1/keys
```

```bash
curl -X POST https://api.vaultwares.com/v1/keys \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "label": "App-Signing-Key-v2",
    "type": "ecc",
    "curve": "P-256",
    "usage": ["sign", "verify"],
    "extractable": false,
    "hsm": "production-hsm"
  }'
```

### Rotationner une clé

```bash
curl -X POST https://api.vaultwares.com/v1/keys/key_abc123/rotate \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -d '{"keep_old_version": true, "deprecate_old_after_days": 90}'
```

## Certificats

### Lister les certificats

```http
GET /v1/certificates
```

### Émettre un certificat

```http
POST /v1/certificates
```

```bash
curl -X POST https://api.vaultwares.com/v1/certificates \
  -H "X-API-Key: vw_live_1234567890abcdef" \
  -H "Content-Type: application/json" \
  -d '{
    "common_name": "api.example.com",
    "sans": ["api.example.com", "api-v2.example.com"],
    "key_label": "TLS-API-Key",
    "validity_days": 365,
    "ca": "internal-ca"
  }'
```

## Réponses d'erreur

| HTTP Status | Error code | Description |
|---|---|---|
| 400 | `invalid_request` | Paramètre de requête manquant ou mal formé |
| 401 | `invalid_api_key` | La clé API est invalide ou révoquée |
| 403 | `insufficient_permissions` | La clé API ne possède pas la permission requise |
| 404 | `not_found` | La ressource demandée n'existe pas |
| 409 | `conflict` | La ressource existe déjà ou conflit d'état |
| 422 | `validation_error` | Le corps de la requête a échoué à la validation |
| 429 | `rate_limit_exceeded` | Trop de requêtes |
| 500 | `internal_error` | Erreur serveur — contactez le support |
