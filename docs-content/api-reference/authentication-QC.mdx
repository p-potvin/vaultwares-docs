## Aperçu

L'API VaultWares utilise des clés API pour l'authentification. Toutes les requêtes API doivent inclure une clé API valide dans les en-têtes de la requête. Les clés API sont associées à votre organisation et peuvent être limitées à des permissions spécifiques.

## Obtenir une clé API

<Steps>
  <Step title="Se connecter au portail">
    Accédez à [portal.vaultwares.com](https://portal.vaultwares.com) et connectez-vous avec vos identifiants.
  </Step>
  <Step title="Accéder aux paramètres API">
    Allez dans **Paramètres** > **Clés API** dans la barre latérale gauche.
  </Step>
  <Step title="Créer une nouvelle clé API">
    Cliquez sur **Créer une clé API** et fournissez :
    - Nom de la clé (pour identification)
    - Permissions (lecture seule, lecture-écriture, administrateur)
    - Date d'expiration (optionnel)
  </Step>
  <Step title="Sauvegarder votre clé">
    Copiez la clé API immédiatement. Elle ne sera affichée qu'une seule fois pour des raisons de sécurité.
  </Step>
</Steps>

<Warning>
  Stockez votre clé API en toute sécurité. Ne jamais committer de clés API dans un système de contrôle de version ni les partager publiquement.
</Warning>

## Méthodes d'authentification

### Clé API dans l'en-tête (recommandé)

Incluez votre clé API dans l'en-tête `X-API-Key`:

```bash
curl https://api.vaultwares.com/v1/devices \
  -H "X-API-Key: vw_live_1234567890abcdef"
```

### Jeton Bearer

Alternativement, utilisez l'en-tête `Authorization` avec un jeton Bearer :

```bash
curl https://api.vaultwares.com/v1/devices \
  -H "Authorization: Bearer vw_live_1234567890abcdef"
```

## Types de clés API

<Tabs>
  <Tab title="Clés actives">
    **Préfixe**: `vw_live_`

    - Utilisé pour les environnements de production
    - Accès aux données et aux appareils réels
    - Des limites de débit s'appliquent
    - Toutes les opérations sont permanentes

    ```bash
    X-API-Key: vw_live_1234567890abcdef
    ```
  </Tab>

  <Tab title="Clés de test">
    **Préfixe**: `vw_test_`

    - Utilisé pour le développement et les tests
    - Accès à l'environnement bac à sable
    - Des limites de débit plus élevées
    - Les données sont isolées de la production

    ```bash
    X-API-Key: vw_test_1234567890abcdef
    ```
  </Tab>

  <Tab title="Clés restreintes">
    **Préfixe**: `vw_restricted_`

    - Limité à des opérations spécifiques
    - Accès en lecture seule
    - Ne peut pas modifier ou supprimer des ressources
    - Utile pour la surveillance et les rapports

    ```bash
    X-API-Key: vw_restricted_1234567890abcdef
    ```
  </Tab>
</Tabs>

## Permissions

Les clés API peuvent être limitées à des permissions spécifiques :

| Permission | Description | Opérations |
|------------|-------------|------------|
| `devices:read` | Voir les informations sur les appareils | GET /devices, GET /devices/:id |
| `devices:write` | Gérer les appareils | POST, PUT, PATCH /devices |
| `devices:delete` | Supprimer les appareils | DELETE /devices/:id |
| `policies:read` | Voir les politiques | GET /policies |
| `policies:write` | Gérer les politiques | POST, PUT, PATCH /policies |
| `users:read` | Voir les utilisateurs | GET /users |
| `users:write` | Gérer les utilisateurs | POST, PUT, PATCH /users |
| `audit:read` | Voir les journaux d'audit | GET /audit |
| `admin` | Accès complet | Toutes les opérations |

### Créer des clés API limitées

```bash
curl https://api.vaultwares.com/v1/api-keys \
  -H "X-API-Key: vw_live_admin_key" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Monitoring Key",
    "permissions": ["devices:read", "audit:read"],
    "expires_at": "2025-12-31T23:59:59Z"
  }'
```

## OAuth 2.0 (Entreprise)

Les clients de niveau entreprise peuvent utiliser OAuth 2.0 pour l'accès délégué par l'utilisateur.

### Flux de code d'autorisation

<Steps>
  <Step title="Rediriger l'utilisateur vers l'URL d'autorisation">
    ```
    https://auth.vaultwares.com/oauth/authorize?
      client_id=your_client_id&
      redirect_uri=https://yourapp.com/callback&
      response_type=code&
      scope=devices:read+devices:write
    ```
  </Step>
  <Step title="L'utilisateur autorise l'application">
    L'utilisateur se connecte et accorde les permissions à votre application.
  </Step>
  <Step title="Recevoir le code d'autorisation">
    VaultWares redirige vers votre URL de rappel avec un code d'autorisation :
    ```
    https://yourapp.com/callback?code=AUTH_CODE
    ```
  </Step>
  <Step title="Échanger le code contre un jeton d'accès">
    ```bash
    curl https://auth.vaultwares.com/oauth/token \
      -d grant_type=authorization_code \
      -d code=AUTH_CODE \
      -d client_id=your_client_id \
      -d client_secret=your_client_secret \
      -d redirect_uri=https://yourapp.com/callback
    ```
  </Step>
  <Step title="Utiliser le jeton d'accès">
    ```bash
    curl https://api.vaultwares.com/v1/devices \
      -H "Authorization: Bearer ACCESS_TOKEN"
    ```
  </Step>
</Steps>

### Jetons de rafraîchissement

Les jetons d'accès expirent après 1 heure. Utilisez des jetons de rafraîchissement pour obtenir de nouveaux jetons d'accès :

```bash
curl https://auth.vaultwares.com/oauth/token \
  -d grant_type=refresh_token \
  -d refresh_token=REFRESH_TOKEN \
  -d client_id=your_client_id \
  -d client_secret=your_client_secret
```

## Limitation de débit

Les requêtes API sont limitées pour assurer une utilisation équitable et la stabilité du système.

### Niveaux de limitation de débit

| Niveau | Requêtes par minute | Requêtes par heure |
|------|------------------|--------------|
| Gratuit | 60 | 1 000 |
| Professionnel | 300 | 10 000 |
| Entreprise | 1 000 | 100 000 |
| Personnalisé | Négociable | Négociable |

### En-têtes de limitation de débit

Chaque réponse API inclut des informations sur la limitation de débit :

```http
HTTP/1.1 200 OK
X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 987
X-RateLimit-Reset: 1640995200
```

- `X-RateLimit-Limit`: Nombre maximum de requêtes autorisé dans la fenêtre actuelle
- `X-RateLimit-Remaining`: Requêtes restantes dans la fenêtre actuelle
- `X-RateLimit-Reset`: Horodatage Unix lorsque la limitation de débit est réinitialisée

### Gérer les limitations de débit

Lorsque vous dépassez la limite de débit, vous recevrez une réponse `429 Too Many Requests` :

```json
{
  "error": {
    "code": "rate_limit_exceeded",
    "message": "Rate limit exceeded. Please retry after 60 seconds.",
    "retry_after": 60
  }
}
```

Implémentez le *backoff* exponentiel dans votre application :

```python
import time
import requests

def make_request_with_retry(url, headers, max_retries=3):
    for attempt in range(max_retries):
        response = requests.get(url, headers=headers)

        if response.status_code == 429:
            retry_after = int(response.headers.get('Retry-After', 60))
            time.sleep(retry_after)
            continue

        return response

    raise Exception("Max retries exceeded")
```

## Bonnes pratiques de sécurité

### Rotation régulière des clés API

```bash
# Create new API key
NEW_KEY=$(curl https://api.vaultwares.com/v1/api-keys \
  -H "X-API-Key: vw_live_current_key" \
  -d '{"name": "Rotated Key", "permissions": ["admin"]}' \
  | jq -r '.key')

# Update your application to use new key
# ...

# Delete old API key
curl -X DELETE https://api.vaultwares.com/v1/api-keys/old_key_id \
  -H "X-API-Key: $NEW_KEY"
```

### Utiliser des variables d'environnement

Ne jamais coder en dur les clés API dans votre application :

```bash
# .env file
VAULTWARES_API_KEY=vw_live_1234567890abcdef
```

```python
import os
from vaultwares import VaultWaresAPI

api_key = os.environ.get('VAULTWARES_API_KEY')
client = VaultWaresAPI(api_key=api_key)
```

### Restreindre par adresse IP

Limitez l'utilisation de la clé API à des adresses IP spécifiques :

```bash
curl https://api.vaultwares.com/v1/api-keys/key_id \
  -X PATCH \
  -H "X-API-Key: vw_live_admin_key" \
  -d '{
    "allowed_ips": ["203.0.113.0/24", "198.51.100.10"]
  }'
```

### Surveiller l'utilisation des clés API

```bash
# Get API key usage statistics
curl https://api.vaultwares.com/v1/api-keys/key_id/usage \
  -H "X-API-Key: vw_live_admin_key"

# Response:
{
  "key_id": "key_abc123",
  "requests_today": 1247,
  "requests_this_month": 45678,
  "last_used_at": "2024-01-15T14:32:10Z",
  "last_used_ip": "203.0.113.42"
}
```

## Signatures de Webhook

Vérifiez l'authenticité des webhooks à l'aide de signatures HMAC.

### Vérifier la signature du webhook

```python
import hmac
import hashlib

def verify_webhook(payload, signature, secret):
    expected_signature = hmac.new(
        secret.encode(),
        payload.encode(),
        hashlib.sha256
    ).hexdigest()

    return hmac.compare_digest(signature, expected_signature)

# Usage
webhook_secret = os.environ.get('VAULTWARES_WEBHOOK_SECRET')
payload = request.body
signature = request.headers.get('X-VaultWares-Signature')

if verify_webhook(payload, signature, webhook_secret):
    # Process webhook
    pass
else:
    # Reject webhook
    return 401
```

## Tester l'authentification

### Tester la validité de la clé API

```bash
curl https://api.vaultwares.com/v1/auth/verify \
  -H "X-API-Key: vw_live_1234567890abcdef"

# Response:
{
  "valid": true,
  "key_id": "key_abc123",
  "permissions": ["admin"],
  "organization_id": "org_xyz789",
  "expires_at": null
}
```

### Tester le jeton OAuth

```bash
curl https://api.vaultwares.com/v1/auth/verify \
  -H "Authorization: Bearer ACCESS_TOKEN"

# Response:
{
  "valid": true,
  "user_id": "user_123",
  "scopes": ["devices:read", "devices:write"],
  "expires_at": "2024-01-15T15:30:00Z"
}
```

## Réponses d'erreur

### Erreurs d'authentification

```json
{
  "error": {
    "code": "invalid_api_key",
    "message": "The provided API key is invalid or has been revoked.",
    "type": "authentication_error"
  }
}
```

Codes d'erreur d'authentification courants :

| Code | Statut HTTP | Description |
|------|-------------|-------------|
| `invalid_api_key` | 401 | La clé API est invalide ou révoquée |
| `expired_api_key` | 401 | La clé API a expiré |
| `insufficient_permissions` | 403 | La clé API ne possède pas les permissions requises |
| `ip_not_allowed` | 403 | Requête provenant d'une adresse IP non autorisée |
| `rate_limit_exceeded` | 429 | Trop de requêtes |
