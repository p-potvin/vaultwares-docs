## Vue d'ensemble

Les exigences de résidence des données imposent que certaines catégories de données soient stockées et traitées uniquement au sein de frontières géographiques spécifiques. Les produits VaultWares fournissent les contrôles techniques pour faire respecter la résidence des données, soutenir les exigences de souveraineté et permettre la preuve d'audit de la conformité.

## Contrôles de résidence des données

### Application sur site (On-premises)

Pour les organisations ayant des exigences de souveraineté strictes, le déploiement sur site de VaultWares garantit que les données ne quittent jamais votre environnement contrôlé :

- **VaultHSM** (PCIe et Réseau) : Tout le matériel de clé reste sur site. Les clés HSM ne sont jamais transmises à VaultWares ou aux fournisseurs de cloud.
- **VaultCrypt** : Les opérations de chiffrement se déroulent localement sur le point d'accès. Aucune donnée n'est transmise aux services cloud sans configuration explicite.
- **VaultBackup** : Les référentiels de sauvegarde peuvent être limités aux NAS sur site et aux installations de co-localisation.
- **VaultAccess** : Déployable entièrement sur site sans dépendances cloud. Intégration AD/LDAP via des contrôleurs de domaine locaux.

### Sélection de région dans le cloud

Pour les organisations utilisant les fonctionnalités cloud de VaultWares, la sélection de région garantit que les données restent dans les limites requises :

<Tabs>
  <Tab title="VaultHSM Cloud">
    ```bash
    # Configure VaultHSM Cloud to EU-only regions
    vw-admin hsm-cloud configure \
      --provider aws \
      --allowed-regions eu-west-1,eu-central-1,eu-west-3 \
      --deny-data-transfer-outside-eu true \
      --sovereign-mode eu-gdpr

    # Verify region restriction
    vw-admin hsm-cloud status --show-regions
    # Output:
    # Active regions: eu-west-1 (Ireland), eu-central-1 (Frankfurt)
    # Data transfer outside EU: DENIED
    # Sovereign mode: EU-GDPR
    ```
  </Tab>
  <Tab title="VaultBackup Cloud">
    ```bash
    # AWS S3: EU-only backup with explicit region lock
    vw-backup repo add \
      --name eu-backup-primary \
      --type s3 \
      --bucket corp-backup-eu-west-1 \
      --region eu-west-1 \
      --restrict-to-region true \
      --deny-cross-region-replication true \
      --encrypt-key hsm:Backup-Master-Key

    # Azure: Germany-only sovereign region
    vw-backup repo add \
      --name de-backup-sovereign \
      --type azure-blob \
      --account corpbackupde \
      --container vaultbackup \
      --region germanynorth \
      --sovereign true
    ```
  </Tab>
  <Tab title="VaultAccess / VaultMonitor">
    ```bash
    # Restrict VaultAccess and VaultMonitor to in-region infrastructure
    vw-admin config set \
      --key data_residency_region \
      --value eu \
      --key audit_log_storage_region \
      --value eu-central-1 \
      --key deny_cross_region_data_flow \
      --value true
    ```
  </Tab>

## Cadres de conformité régionaux

<AccordionGroup>
  <Accordion title="Union européenne (RGPD, Frontière de données UE)">
    **Exigences** : Les données personnelles des résidents de l'UE ne doivent pas être transférées en dehors de l'UE/EEE sans garanties adéquates (Chapitre V du RGPD).

    **Contrôles VaultWares** :
    - VaultHSM Cloud prend en charge le stockage de clés uniquement dans l'UE (régions de Dublin, Francfort, Paris)
    - VaultBackup prend en charge les régions S3 et Azure Blob uniquement dans l'UE avec des verrous stricts
    - Tous les journaux d'audit peuvent être limités au stockage de l'UE
    - Les Clauses Contractuelles Types (CCT) sont disponibles pour les services VaultWares Cloud

    ```bash
    # Enable EU data boundary enforcement
    vw-admin data-residency enable \
      --region EU \
      --standard GDPR \
      --deny-non-compliant-transfers true \
      --alert-on-violation true \
      --log-all-transfers true
    ```
  </Accordion>

  <Accordion title="Royaume-Uni (UK GDPR, Data Protection Act 2018)">
    **Exigences** : Similaire au RGPD de l'UE après le Brexit. Les transferts de données vers des pays non adéquats nécessitent des garanties. VaultWares maintient des options de résidence des données spécifiques au Royaume-Uni.

    ```bash
    vw-admin data-residency enable \
      --region UK \
      --standard UK-GDPR \
      --allowed-cloud-regions eu-west-2 \  # London
      --deny-non-compliant-transfers true
    ```
  </Accordion>

  <Accordion title="Allemagne (C5, BDSG, Cloud Computing Compliance)">
    **Exigences** : Attestation BSI C5 et exigences de la Loi fédérale sur la protection des données pour le secteur public et les institutions financières.

    VaultWares prend en charge :
    - Le déploiement dans des régions cloud souveraines allemandes (Azure Germany North/West Central, Deutsche Telekom T-Systems)
    - Toutes les données et clés traitées exclusivement sur l'infrastructure allemande lorsqu'elles sont configurées
    - Preuve d'audit BSI C5 à partir des rapports VaultMonitor
  </Accordion>

  <Accordion title="États-Unis (FedRAMP, StateRAMP, ITAR)">
    **Exigences** : FedRAMP exige que les services cloud utilisés par les agences fédérales soient hébergés dans des régions américaines. ITAR restreint certaines données techniques aux personnes et installations américaines.

    ```bash
    # Restrict to US GovCloud regions (FedRAMP)
    vw-admin data-residency enable \
      --region US-GOV \
      --standard FedRAMP \
      --allowed-cloud-regions us-gov-west-1,us-gov-east-1 \
      --restrict-to-us-persons true
    ```
  </Accordion>

  <Accordion title="Australie (Privacy Act, CDR, APRA CPS 234)">
    **Exigences** : Les entités réglementées par l'APRA (banques, assureurs, fonds de retraite) doivent s'assurer que les données ne sont pas transférées à l'étranger sans contrôles appropriés.

    ```bash
    vw-admin data-residency enable \
      --region AU \
      --standard APRA-CPS234 \
      --allowed-cloud-regions ap-southeast-2 \  # Sydney
      --deny-non-compliant-transfers true
    ```
  </Accordion>

  <Accordion title="Chine (PIPL, MLPS, DSL)">
    **Exigences** : La Loi chinoise sur la protection des informations personnelles exige que les données personnelles des résidents chinois soient stockées en Chine, avec des contrôles stricts sur les transferts transfrontaliers.

    VaultWares prend en charge un déploiement sur site entièrement isolé pour les exigences de souveraineté des données chinoises. Les fonctionnalités cloud ne sont pas disponibles pour les données régies par la PIPL sans évaluation supplémentaire.
  </Accordion>
</AccordionGroup>

## Transferts internationaux de données RGPD

Lorsque les données doivent être transférées en dehors de l'UE/EEE, le Chapitre V du RGPD exige l'un des éléments suivants :

| Mécanisme de transfert | Support VaultWares |
|---|---|
| Décision d'adéquation (ex. : Royaume-Uni, Japon, Israël) | Automatiquement autorisé ; VaultWares n'applique aucune restriction supplémentaire |
| Clauses Contractuelles Types (CCT) | VaultWares exécute les CCT pour le traitement des données des services cloud |
| Règles d'entreprise contraignantes (REC) | Pris en charge pour les clients d'entreprise avec des programmes REC |
| Consentement explicite | Configurables par catégorie de données dans les politiques ABAC de VaultAccess |

```bash
# Tag data categories for transfer restriction
vw-admin data-classification tag \
  --directory /data/personal \
  --classification personal-data-eu \
  --transfer-restriction gdpr-eu-only \
  --audit-tag true

# Enforce transfer restrictions via VaultGate policy
vaultgate data-transfer-policy create \
  --name Deny-EU-PII-Outside-EU \
  --source-classification personal-data-eu \
  --deny-destination-regions outside-eu \
  --log-violations true \
  --alert-on-violation true
```

## Preuve d'audit

VaultWares fournit une preuve d'audit complète pour la conformité de résidence des données :

```bash
# Generate data residency compliance report
vw-admin report data-residency \
  --standard GDPR \
  --start "2024-10-01" \
  --end "2024-10-31" \
  --include-storage-locations true \
  --include-transfer-logs true \
  --include-key-storage-locations true \
  --format pdf \
  --output /reports/data-residency-oct-2024.pdf

# Verify no cross-boundary transfers in last 30 days
vw-admin data-residency audit \
  --since 30d \
  --check-transfers true \
  --report-violations
```
