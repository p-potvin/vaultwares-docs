## Vue d'ensemble

Le Règlement général sur la protection des données (RGPD) exige que les organisations qui traitent des données personnelles de résidents de l'UE/EEE mettent en œuvre des mesures techniques et organisationnelles appropriées pour protéger ces données. Les produits VaultWares fournissent les contrôles techniques nécessaires pour répondre aux exigences de sécurité du RGPD.

<Note>
  Ce guide fournit des conseils techniques sur la manière dont les produits VaultWares soutiennent la conformité au RGPD. Il ne constitue pas un avis juridique. Consultez votre Responsable de la protection des données (RPD) et un conseiller juridique pour une évaluation complète de la conformité.
</Note>

## Articles pertinents du RGPD

<AccordionGroup>
  <Accordion title="Article 5 — Principes de traitement des données">
    **Pertinence** : Le principe d'intégrité et de confidentialité (Art. 5(1)(f)) exige que les données personnelles soient traitées « d'une manière qui assure la sécurité appropriée des données personnelles, y compris la protection contre le traitement non autorisé ou illicite et contre la perte, la destruction ou les dommages accidentels. »

    **Contrôles VaultWares** :
    - VaultDrive et VaultCrypt assurent la confidentialité des données au repos
    - VaultHSM fournit la gestion des clés pour protéger les clés de chiffrement
    - VaultBackup assure la disponibilité des données et la reprise après sinistre
  </Accordion>
  <Accordion title="Article 25 — Protection des données dès la conception et par défaut">
    **Pertinence** : Exige la mise en œuvre de mesures de protection des données dès le départ, et non après coup.

    **Contrôles VaultWares** :
    - Le chiffrement matériel sur VaultDrive est toujours actif; il ne peut pas être désactivé
    - VaultAccess applique par défaut le moindre privilège; l'accès doit être explicitement accordé
    - VaultCrypt peut appliquer des volumes toujours chiffrés via une politique GPO/MDM
  </Accordion>
  <Accordion title="Article 32 — Sécurité du traitement">
    **Pertinence** : Exige des « mesures techniques et organisationnelles appropriées », y compris le chiffrement, la pseudonymisation, l'assurance continue de la confidentialité/intégrité/disponibilité, et des tests réguliers.

    **Contrôles VaultWares** :
    - Chiffrement AES-256-XTS pour le stockage; TLS 1.3 pour la transmission
    - Gestion des clés VaultHSM certifiée FIPS 140-3 Niveau 3/4
    - VaultMonitor pour la surveillance continue de la sécurité et la détection d'anomalies
    - VaultBackup pour les tests automatisés de récupération
  </Accordion>
  <Accordion title="Article 17 — Droit à l'effacement ('Droit à l'oubli')">
    **Pertinence** : Les personnes concernées peuvent demander la suppression de leurs données personnelles.

    **Contrôles VaultWares** :
    La capacité d'effacement cryptographique de VaultCrypt rend les données définitivement inaccessibles en détruisant la clé de chiffrement, ce qui est généralement plus rapide et plus fiable que d'écraser les données (surtout sur les SSD).

    ```bash
    # Cryptographic erasure of a VaultCrypt volume
    vaultcrypt crypto-erase \
      --volume /dev/sdc \
      --confirm \
      --audit-log "GDPR right-to-erasure request #DSR-2024-1115"

    # Remote cryptographic erasure of a VaultDrive
    vw-storage crypto-erase \
      --device VD-ENT-00247 \
      --reason "GDPR erasure request" \
      --audit-log
    ```
  </Accordion>
  <Accordion title="Article 20 — Droit à la portabilité des données">
    **Pertinence** : Les personnes concernées peuvent demander leurs données dans un format portable.

    **Contrôles VaultWares** :
    VaultAccess et VaultMonitor fournissent des API d'exportation de données qui permettent aux organisations d'extraire des données spécifiques à l'utilisateur dans des formats standard (JSON, CSV) pour les demandes d'accès des personnes concernées.

    ```bash
    curl https://api.vaultwares.com/v1/users/user_abc123/data-export \
      -H "X-API-Key: vw_live_..." \
      -d '{"format": "json", "include": ["access_logs", "profile", "preferences"]}'
    ```
  </Accordion>
  <Accordion title="Article 33 — Notification d'une violation de données personnelles">
    **Pertinence** : Les organisations doivent notifier l'autorité de contrôle dans les 72 heures suivant la prise de connaissance d'une violation.

    **Contrôles VaultWares** :
    VaultMonitor peut détecter et alerter sur des indicateurs de violation de données suspectés (accès non autorisé à des données en masse, téléchargements anormaux, compromission d'identifiants) en quelques minutes. L'alerte automatisée et l'exportation de journaux forensiques soutiennent la fenêtre de notification de 72 heures.
  </Accordion>
</AccordionGroup>

## Implémentation technique

### Configuration de chiffrement pour le RGPD

```bash
# VaultCrypt: Enable mandatory encryption for all drives in scope
vaultcrypt policy set \
  --name GDPR-Personal-Data \
  --algorithm AES-256-XTS \
  --key-source hsm \
  --require-pre-boot-auth true \
  --apply-to-volumes /data/personal-data \
  --audit-log-all-access true

# VaultDrive: Enforce corporate policy including audit logging
vw-admin deploy-config \
  --policy gdpr-compliant-storage \
  --device-group EU-PersonalData-Drives \
  --require-2fa true \
  --audit-logging enabled \
  --geofencing eu-countries
```

### Journalisation d'audit pour le RGPD

Le principe de responsabilité du RGPD (Art. 5(2)) exige que les organisations démontrent leur conformité. Les journaux d'audit inviolables de VaultMonitor soutiennent cela :

```bash
# Enable comprehensive GDPR audit logging
vw-admin audit configure \
  --scope personal-data-access \
  --log-level detailed \
  --include-user-identity true \
  --include-resource-accessed true \
  --include-source-ip true \
  --retention-days 1825 \  # 5 years
  --sign-with hsm:AuditLog-Signing-Key \
  --export-to-siem true

# Generate a Data Subject Access Request (DSAR) report
vw-admin dsar-report \
  --subject-email alice@example.com \
  --start "2024-01-01" \
  --end "2024-11-15" \
  --output /reports/dsar-alice-$(date +%Y%m%d).pdf
```

### Contrôles de résidence des données

Pour la conformité au RGPD concernant les transferts de données en dehors de l'UE/EEE :

```bash
# Restrict VaultHSM Cloud key storage to EU regions
vw-admin cloud configure \
  --key-storage-regions eu-west-1,eu-central-1 \
  --deny-non-eu-regions true

# Configure VaultBackup to keep data in EU regions only
vw-backup repo add \
  --name eu-cloud-backup \
  --type s3 \
  --bucket corp-vaultbackup-eu \
  --region eu-central-1 \
  --restrict-to-region true
```

## Liste de contrôle de conformité au RGPD

<AccordionGroup>
  <Accordion title="Contrôles techniques">
    - [ ] Toutes les données personnelles chiffrées au repos (VaultDrive ou VaultCrypt)
    - [ ] Toutes les données personnelles chiffrées en transit (TLS 1.3)
    - [ ] Les clés de chiffrement gérées par VaultHSM (pas de coffres-forts logiciels)
    - [ ] Capacité d'effacement cryptographique testée et documentée
    - [ ] La journalisation d'audit est activée pour tout accès aux données personnelles
    - [ ] Les journaux d'audit sont protégés contre la falsification (signés par HSM)
    - [ ] Rétention des journaux d'audit ≥ 5 ans (configurable dans VaultMonitor)
  </Accordion>
  <Accordion title="Contrôle d'accès">
    - [ ] Le moindre privilège est appliqué via VaultAccess RBAC
    - [ ] L'authentification multifacteur (MFA) est requise pour l'accès aux systèmes de données personnelles
    - [ ] Le PAM est en place pour l'accès privilégié aux bases de données contenant des données personnelles
    - [ ] Les examens d'accès sont effectués trimestriellement
    - [ ] Le départ automatisé révoque l'accès dans les 24 heures
  </Accordion>
  <Accordion title="Droits des personnes concernées">
    - [ ] Procédure d'exportation des demandes d'accès (DSAR) documentée et testée
    - [ ] Procédure d'effacement cryptographique documentée et testée
    - [ ] Exportation de portabilité des données dans un format lisible par machine disponible
    - [ ] SLA pour répondre aux DSAR définis (RGPD : 1 mois)
  </Accordion>
  <Accordion title="Préparation à la notification de violation">
    - [ ] Les règles d'alerte de VaultMonitor couvrent les indicateurs de violation
    - [ ] Le guide de réponse en 72 heures est documenté et répété
    - [ ] La liste de contacts en cas d'incident est à jour
    - [ ] La procédure d'exportation de journaux forensiques est testée
  </Accordion>
</AccordionGroup>
