## Overview

Data residency requirements mandate that certain categories of data be stored and processed only within specific geographic boundaries. VaultWares products provide the technical controls to enforce data residency, support sovereignty requirements, and enable audit evidence of compliance.

## Data residency controls

### On-premises enforcement

For organizations with strict sovereignty requirements, VaultWares' on-premises deployment ensures data never leaves your controlled environment:

- **VaultHSM** (PCIe and Network): All key material remains on-premises. HSM keys are never transmitted to VaultWares or cloud providers.
- **VaultCrypt**: Encryption operations occur locally on the endpoint. No data is transmitted to cloud services without explicit configuration.
- **VaultBackup**: Backup repositories can be restricted to on-premises NAS and co-location facilities.
- **VaultAccess**: Deployable fully on-premises with no cloud dependencies. AD/LDAP integration via local domain controllers.

### Cloud region selection

For organizations using VaultWares cloud features, region selection ensures data stays within required boundaries:

<Tabs>
  <Tab title="VaultHSM Cloud">
    ```bash
    # Configure VaultHSM Cloud to EU-only regions
    vw-admin hsm-cloud configure \
      --provider aws \
      --allowed-regions eu-west-1,eu-central-1,eu-west-3 \
      --deny-data-transfer-outside-eu true \
      --sovereign-mode eu-gdpr

    # Verify region restriction
    vw-admin hsm-cloud status --show-regions
    # Output:
    # Active regions: eu-west-1 (Ireland), eu-central-1 (Frankfurt)
    # Data transfer outside EU: DENIED
    # Sovereign mode: EU-GDPR
    ```
  </Tab>
  <Tab title="VaultBackup Cloud">
    ```bash
    # AWS S3: EU-only backup with explicit region lock
    vw-backup repo add \
      --name eu-backup-primary \
      --type s3 \
      --bucket corp-backup-eu-west-1 \
      --region eu-west-1 \
      --restrict-to-region true \
      --deny-cross-region-replication true \
      --encrypt-key hsm:Backup-Master-Key

    # Azure: Germany-only sovereign region
    vw-backup repo add \
      --name de-backup-sovereign \
      --type azure-blob \
      --account corpbackupde \
      --container vaultbackup \
      --region germanynorth \
      --sovereign true
    ```
  </Tab>
  <Tab title="VaultAccess / VaultMonitor">
    ```bash
    # Restrict VaultAccess and VaultMonitor to in-region infrastructure
    vw-admin config set \
      --key data_residency_region \
      --value eu \
      --key audit_log_storage_region \
      --value eu-central-1 \
      --key deny_cross_region_data_flow \
      --value true
    ```
  </Tab>

## Regional compliance frameworks

<AccordionGroup>
  <Accordion title="European Union (GDPR, EU Data Boundary)">
    **Requirements**: Personal data of EU residents must not be transferred outside the EU/EEA without adequate safeguards (GDPR Chapter V).

    **VaultWares controls**:
    - VaultHSM Cloud supports EU-only key storage (Dublin, Frankfurt, Paris regions)
    - VaultBackup supports EU-only S3 and Azure Blob regions with hard locks
    - All audit logs can be restricted to EU storage
    - Standard Contractual Clauses (SCCs) available for VaultWares Cloud services

    ```bash
    # Enable EU data boundary enforcement
    vw-admin data-residency enable \
      --region EU \
      --standard GDPR \
      --deny-non-compliant-transfers true \
      --alert-on-violation true \
      --log-all-transfers true
    ```
  </Accordion>

  <Accordion title="United Kingdom (UK GDPR, Data Protection Act 2018)">
    **Requirements**: Similar to EU GDPR following Brexit. Data transfers to non-adequate countries require safeguards. VaultWares maintains UK-specific data residency options.

    ```bash
    vw-admin data-residency enable \
      --region UK \
      --standard UK-GDPR \
      --allowed-cloud-regions eu-west-2 \  # London
      --deny-non-compliant-transfers true
    ```
  </Accordion>

  <Accordion title="Germany (C5, BDSG, Cloud Computing Compliance)">
    **Requirements**: German BSI C5 attestation and Federal Data Protection Act requirements for public sector and financial institutions.

    VaultWares supports:
    - Deployment in German sovereign cloud regions (Azure Germany North/West Central, Deutsche Telekom T-Systems)
    - All data and keys processed exclusively on German infrastructure when configured
    - BSI C5 audit evidence from VaultMonitor reports
  </Accordion>

  <Accordion title="United States (FedRAMP, StateRAMP, ITAR)">
    **Requirements**: FedRAMP requires cloud services used by federal agencies to be hosted in US regions. ITAR restricts certain technical data to US persons and facilities.

    ```bash
    # Restrict to US GovCloud regions (FedRAMP)
    vw-admin data-residency enable \
      --region US-GOV \
      --standard FedRAMP \
      --allowed-cloud-regions us-gov-west-1,us-gov-east-1 \
      --restrict-to-us-persons true
    ```
  </Accordion>

  <Accordion title="Australia (Privacy Act, CDR, APRA CPS 234)">
    **Requirements**: APRA-regulated entities (banks, insurers, super funds) must ensure data is not transferred offshore without appropriate controls.

    ```bash
    vw-admin data-residency enable \
      --region AU \
      --standard APRA-CPS234 \
      --allowed-cloud-regions ap-southeast-2 \  # Sydney
      --deny-non-compliant-transfers true
    ```
  </Accordion>

  <Accordion title="China (PIPL, MLPS, DSL)">
    **Requirements**: China's Personal Information Protection Law requires personal data of Chinese residents to be stored in China, with strict controls on cross-border transfers.

    VaultWares supports fully air-gapped on-premises deployment for Chinese data sovereignty requirements. Cloud features are not available for data governed by PIPL without additional assessment.
  </Accordion>
</AccordionGroup>

## GDPR international data transfers

When data must transfer outside the EU/EEA, GDPR Chapter V requires one of:

| Transfer mechanism | VaultWares support |
|---|---|
| Adequacy decision (e.g., UK, Japan, Israel) | Automatically permitted; VaultWares applies no additional restrictions |
| Standard Contractual Clauses (SCCs) | VaultWares executes SCCs for cloud services data processing |
| Binding Corporate Rules (BCRs) | Supported for enterprise customers with BCR programmes |
| Explicit consent | Configurable per data category in VaultAccess ABAC policies |

```bash
# Tag data categories for transfer restriction
vw-admin data-classification tag \
  --directory /data/personal \
  --classification personal-data-eu \
  --transfer-restriction gdpr-eu-only \
  --audit-tag true

# Enforce transfer restrictions via VaultGate policy
vaultgate data-transfer-policy create \
  --name Deny-EU-PII-Outside-EU \
  --source-classification personal-data-eu \
  --deny-destination-regions outside-eu \
  --log-violations true \
  --alert-on-violation true
```

## Audit evidence

VaultWares provides comprehensive audit evidence for data residency compliance:

```bash
# Generate data residency compliance report
vw-admin report data-residency \
  --standard GDPR \
  --start "2024-10-01" \
  --end "2024-10-31" \
  --include-storage-locations true \
  --include-transfer-logs true \
  --include-key-storage-locations true \
  --format pdf \
  --output /reports/data-residency-oct-2024.pdf

# Verify no cross-boundary transfers in last 30 days
vw-admin data-residency audit \
  --since 30d \
  --check-transfers true \
  --report-violations
```
